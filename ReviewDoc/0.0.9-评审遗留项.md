# 0.0.9 评审遗留项

## 1. 背景

`0.0.9` 聚焦于相机与聚焦系统的结构整理，目标是显式拆分 `render bounds / focus bounds` 并收紧 `Viewport` 职责。

在本版本结构整理完成后，下一阶段最自然的技术方向已经明确为：

- 正交模式的真实投影路径接入
- `Lens Zoom` 的行为边界与 UI 方案

但这两项不应插入当前版本实现，应作为后续评审输入单独处理。

## 2. 发现时间

- 2026-03-12

## 3. 所属版本

- 作为 `0.0.9` 结束后的后续版本评审输入

## 4. 是否阻塞当前步骤

- 否

说明：

- 当前 `0.0.9` 的结构整理可以独立完成
- 正交模式与 `Lens Zoom` 并不阻塞当前版本闭环

## 5. 风险说明

- 若在结构未稳定前直接接入正交模式，`Camera / OrbitCameraController / Viewport` 的职责会再次耦合
- 若在正交模式真实路径未接通前提前引入 `Lens Zoom`，行为语义和 UI 策略会不稳定
- 若不先明确正交投影参数与状态链，后续 `ProjectionMode / ZoomMode` 的扩展容易返工

## 6. 建议处理版本

- 建议下一版本优先处理正交模式真实投影路径
- `Lens Zoom` 建议再后置一个版本评审

## 7. 建议方案

### 7.1 正交模式真实投影路径

建议优先完成：

1. `Camera` 真正支持正交投影矩阵
2. `orthographicHeight` 接入真实投影参数链
3. `OrbitCameraController -> Camera -> Viewport` 打通正交模式状态
4. 默认行为保持透视模式，先不急于开放 UI 切换

### 7.2 Lens Zoom 行为边界

建议在正交模式稳定后再评审：

1. `Lens Zoom` 在透视模式下是否改 `FOV`
2. `Lens Zoom` 在正交模式下是否改 `orthographicHeight`
3. 是否允许：
   - `Perspective + Dolly`
   - `Perspective + Lens`
   - `Orthographic + Dolly`
   - `Orthographic + Lens`
4. UI 是否需要显式暴露 `ZoomMode`
