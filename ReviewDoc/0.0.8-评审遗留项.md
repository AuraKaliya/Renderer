# 0.0.8 评审遗留项

## 1. 背景

在 `0.0.7` 完成聚焦链路接入后，围绕相机缩放与裁剪行为继续进行了讨论。讨论确认：

- 当前滚轮缩放采用 Dolly 语义
- 缩放过程中若 `near/far` 未联动更新，近裁剪面会更容易切到物体
- 后续还需要考虑 `ProjectionMode / ZoomMode` 的扩展接口

## 2. 发现时间

- 2026-03-12

## 3. 所属版本

- 作为 `0.0.8` 的评审输入

## 4. 是否阻塞当前步骤

- 否

说明：

- 该问题不阻塞 `0.0.7` 当前计划完成
- 但会影响下一阶段相机交互与裁剪策略的可扩展性

## 5. 风险说明

- 若继续保持当前缩放实现而不补 `near/far` 联动，近距离观察时仍可能出现裁剪体验不稳定
- 若不提前拆分 `ProjectionMode / ZoomMode`，后续接入正交模式与双缩放模式时接口容易耦合
- 若继续把裁剪逻辑混在聚焦 helper 中，后续维护成本会快速上升

## 6. 建议处理版本

- 建议纳入 `0.0.8`

## 7. 建议方案

### 7.1 裁剪计算拆层

- 将 `near/far` 计算从当前聚焦 helper 中独立出来
- 建议新增：
  - `camera_clip_utils`

### 7.2 缩放模式状态建模

- 增加：
  - `ProjectionMode`
  - `ZoomMode`
- 建议两者独立建模，不合并为单一枚举

### 7.3 版本实现顺序

建议按以下顺序推进：

1. 先补透视模式下 Dolly 缩放的 `near/far` 联动
2. 再抽 `ProjectionMode / ZoomMode` 状态
3. 最后再考虑正交模式与 Lens Zoom

## 9. 补充遗留项：render bounds / focus bounds 语义拆分

### 9.1 背景

在修复 `Focus All / Focus Sphere` 重复点击不稳定问题时，当前实现引入了“稳定聚焦 bounds”：

- `render bounds`
  - 面向当前真实渲染与瞬时 world AABB
- `focus bounds`
  - 面向聚焦行为的稳定范围

### 9.2 风险

- 当前两种 bounds 语义仍主要收口在 `Viewport` 内部
- 随着后续裁剪、选中、聚焦、调试显示继续扩展，若不显式命名和拆层，后续维护成本会上升

### 9.3 建议

- 后续评审时明确区分：
  - `render bounds`
  - `focus bounds`
- 必要时抽出独立 helper，例如：
  - `camera_focus_bounds_utils`
