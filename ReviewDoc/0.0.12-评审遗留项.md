# 0.0.12 评审遗留项
## 1. 背景

`0.0.11` 已完成：

- 正交模式默认主缩放改为 `Lens`
- 正交模式下的聚焦与缩放行为收口

在继续推进 `Perspective + Lens` 讨论时，又明确了两类后续需求：

1. 鼠标位置驱动的视口缩放
2. 模型参数调整后，在保持当前视角的前提下更新 bounds 与裁剪

这两个需求说明后续必须显式区分：

- `Camera Zoom`
- `Viewport Zoom`
- `Auto Framing / Focus`

## 2. 发现时间

- 2026-03-12

## 3. 所属版本

- 作为 `0.0.12` 评审输入

## 4. 是否阻塞当前步骤

- 否

说明：

- `0.0.11` 已完成当前版本目标
- 这些内容属于下一阶段能力扩展与边界固化，不阻塞当前版本收尾

## 5. 风险说明

- 若在没有显式拆分概念的前提下直接引入 `Perspective + Lens`，后续会与 `Viewport Zoom` 混淆
- 若把“模型参数变化后的 bounds 更新”直接等同于自动 `Focus All`，会破坏“保持当前视角”的需求
- 若没有先明确“保持当前视角 / 自动适配视图”策略，后续参数化建模更新会不断改写相机状态

## 6. 建议处理版本

- 建议 `0.0.12` 优先评审并实现 `Perspective + Lens`
- `Viewport Zoom` 与模型参数变化策略建议继续保留为后续评审输入，不插入 `0.0.12` 首版实现

## 6.1 当前状态

- 状态：已纳入版本 0.0.12
- 对应计划：`VersionDoc/0.0.12-开发计划.md`
- 纳入范围说明：
  - 本版本只吸收 `Perspective + Lens`
  - `Viewport Zoom`
  - 模型参数变化后的策略切换
  继续保留为后续评审输入

## 6.2 后续保留项

- `Viewport Zoom`
- 模型参数变化后的：
  - 保持当前视角
  - 自动适配视图
  策略

## 7. 建议方案

### 7.1 Perspective + Lens

建议优先处理：

1. `Perspective + Lens` 只修改 `verticalFovDegrees`
2. `Perspective + Dolly` 继续作为默认透视主路径
3. `Focus Bounds` 在 `Perspective + Lens` 下仍优先改 `distance`
4. `Focus Point` 在 `Perspective + Lens` 下不自动改 `FOV`

### 7.2 Viewport Zoom

建议后续单独评审：

1. 定义为独立概念，不归入 `Dolly` 或 `Lens`
2. 不改变透视感
3. 不直接改变 `near/far`
4. 以鼠标位置作为屏幕空间锚点

### 7.3 模型参数变化策略

建议后续单独评审：

1. 默认支持两种策略：
   - 保持当前视角
   - 自动适配视图
2. 默认不自动触发重新 framing
3. 模型变化后自动更新：
   - `localBounds`
   - `worldBounds`
   - `sceneBounds`
   - `near/far`
