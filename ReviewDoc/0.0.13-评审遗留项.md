# 0.0.13 评审遗留项

## 1. 背景

`0.0.12` 已完成：

- `Perspective + Lens` 最小闭环
- `Zoom Mode` 的最小验证入口
- 正交模式与透视模式下基础相机行为打通

随着 `Zoom Mode` UI 继续讨论，已明确当前下一阶段的主要问题不再是单一相机行为缺失，而是 UI 层规则表达与职责边界开始变得复杂，尤其集中在：

- `CameraControlWidget`
- `ViewerWindow` 的状态同步与编排

## 2. 发现时间

- 2026-03-12

## 3. 所属版本

- 作为 `0.0.13` 评审输入

## 4. 是否阻塞当前步骤

- 否

说明：

- `0.0.12` 已完成其版本目标
- 当前问题属于下一阶段的结构优化与 UI 规则固化，不阻塞 `0.0.12` 收尾

## 5. 风险说明

- 若继续在当前 `CameraControlWidget` 中叠加规则，后续会使相机 UI 同时承担：
  - 投影模式控制
  - 缩放模式控制
  - active parameter 选择
  - `Focus Point`
  - `Near/Far`
  - `Orbit Center`
- 若继续在 `ViewerWindow` 中堆积同步与编排逻辑，后续接入：
  - `Viewport Zoom`
  - 模型参数变化后的视角策略
  时，会导致窗口层继续膨胀
- 若不先整理 UI 规则，后续用户会看到“参数能改，但不清楚当前哪个参数才是主控制量”的问题

## 6. 建议处理版本

- 建议 `0.0.13` 优先处理

- 对应计划：`VersionDoc/0.0.13-开发计划.md`

- 完成说明：
  - 本版本完成 `Zoom Mode` UI 规则整理
  - 本版本完成 `CameraControlWidget` 的继续拆解
  - 本版本完成 `ViewerWindow` 的 UI 同步/编排收口

## 8. 建议方案

### 8.1 优先目标

`0.0.13` 优先处理“Zoom Mode 的完整 UI 策略”和“UI 层职责继续收紧”，而不是直接继续扩新功能。

### 8.2 建议范围

建议纳入：

1. 固化 `Projection + Zoom Mode + Active Parameter` 的 UI 规则
2. 继续拆解 `CameraControlWidget` 内部职责
3. 收口 `ViewerWindow` 中的 UI 同步/编排职责

建议暂不纳入：

1. `Viewport Zoom`
2. 模型参数变化后的“保持当前视角 / 自动适配视图”策略实现
3. `Orthographic + Dolly` 的 UI 开放

### 8.3 拆解方向

#### CameraControlWidget

建议按内部职责分为两块：

1. `Mode / Parameter Section`
- `Projection`
- `Zoom Mode`
- `Distance`
- `Vertical FOV`
- `Ortho Height`

2. `Focus / State Section`
- `Near/Far`
- `Orbit Center`
- `Focus Point`

#### ViewerWindow

建议逐步抽离出轻量 UI 状态同步层，例如：

- `ViewerUiStateAdapter`
- 或 `ViewerControlBinder`

用于集中处理：

- `Viewport -> Panel` 状态回填
- enable/disable 规则同步
- 当前 active parameter 的界面表达

#### UI 规则

后续建议将相机 UI 规则从控件代码中抽离，形成独立 helper 或 policy 层，例如：

- `camera_mode_ui_policy`

用于统一输出：

- primary 参数
- secondary 参数
- inactive 参数

## 10. 补充遗留项：控制面板承载方式

### 10.1 背景

随着控制参数增加，当前右侧控制面板已经接近固定高度布局的承载上限。

### 10.2 建议

- 下调主窗口默认初始高度
- 为 `ViewerControlPanel` 引入 `QScrollArea`
- 保持现有子面板结构不变

### 10.3 处理建议

- 建议作为后续版本 UI 承载优化输入
- 不插入当前 `0.0.13` 已冻结步骤

### 10.4 当前状态

- 状态：保留为后续评审输入
- 建议处理版本：后续 UI 承载优化版本
