# 0.0.2-开发记录

## 1. 版本目标

将 `test_viewer` 当前集中在 `ViewerWindow::buildControlPanel()` 中的控制面板逻辑拆分为独立 Widget，降低 `ViewerWindow` 复杂度，并建立可继续扩展的 UI 结构。

## 2. 实际完成内容

### 2.1 总面板抽离

- 新增 `ViewerControlPanel`
- `ViewerWindow` 不再直接承载右侧面板实现细节
- `ViewerWindow` 改为只负责：
  - 主窗口布局
  - `Viewport` 与控制面板的信号连接
  - 面板状态同步

### 2.2 光照与相机控件拆分

- 新增 `LightingControlWidget`
- 新增 `CameraControlWidget`
- `ViewerControlPanel` 改为装配这两个子组件并转发信号

### 2.3 对象控件拆分

- 新增 `SceneObjectControlWidget`
- 使用 3 个对象控件实例替换总面板中的重复对象控制逻辑

## 5. 已知问题与限制

- 当前控制面板虽然已模块化，但仍处于测试工具形态，未做进一步样式与交互优化
- 当前相机控制仍以参数面板为主，尚未接入鼠标交互
- 当前材质仍以纯色为主，尚未引入纹理资源句柄和纹理采样链路

## 6. 下一版本建议

建议 `0.0.3` 聚焦于引入 `TextureHandle`：

- 在 `scene_contract` 中建立纹理资源协议
- 在 `render_gl` 中建立纹理资源接口与缓存入口
- 保持 `parametric_model` 与 OpenGL 后端解耦
- 分阶段接入材质纹理采样
