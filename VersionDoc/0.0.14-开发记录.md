# 0.0.14 开发记录
## 1. 版本目标

围绕当前测试界面的控制面板承载压力与相机 UI 规则分散问题，继续收口 `test_viewer` 的 UI 结构，使当前界面在较小窗口尺寸下仍具备稳定可操作性，同时把 `Projection + Zoom Mode + Active Parameter` 的规则从控件实现中进一步抽离，为后续 `Viewport Zoom` 接入提供更清晰的结构边界。

## 2. 实际完成内容

### 2.1 控制面板滚动承载与窗口布局收口

- `ViewerControlPanel` 内部改为：
  - 固定宽度外壳
  - `QScrollArea`
  - 内容容器纵向布局
- 保持现有：
  - Scene Object
  - Lighting
  - Camera
  - Bounds
  - Actions
  子面板结构不变
- 主窗口默认尺寸由 `1280 x 720` 调整为更克制的 `1240 x 680`

### 2.2 相机模式 UI 规则层抽离

- 新增：
  - `src/test_viewer/src/camera_mode_ui_policy.h`
  - `src/test_viewer/src/camera_mode_ui_policy.cpp`
- 将 `Projection + Zoom Mode + Active Parameter` 对应的规则输出统一收口到 `camera_mode_ui_policy`
- `CameraControlWidget` 不再直接持有相机模式规则判断，只负责：
  - 控件展示
  - 角色文本映射
  - 角色样式映射
- 当前以下组合的 UI 启停与主次角色语义继续保持：
  - `Perspective + Dolly`
  - `Perspective + Lens`
  - `Orthographic + Lens`

### 2.3 ViewerWindow 面板同步编排收口

- 在 `ViewerControlPanel` 中新增整体状态结构：
  - `SceneObjectPanelState`
  - `LightingPanelState`
  - `PanelState`
- 新增 `ViewerControlPanel::setPanelState()`，统一承接面板回填
- `Viewport` 新增 `controlPanelState()`，集中组装当前 UI 所需状态
- `ViewerWindow` 删除分散的：
  - `syncSceneObjectPanel()`
  - `syncLightingPanel()`
  - `syncCameraPanel()`
- `syncControlPanel()` 现在只保留为：
  - 从 `Viewport` 取整体状态
  - 一次性回填到 `ViewerControlPanel`
