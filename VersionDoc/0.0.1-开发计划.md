# 0.0.1-开发计划

## 1. 版本目标

本版本聚焦于将 `test_viewer` 中的相机能力从 `Viewport` 内部状态中拆出，形成独立的相机对象与相机控制器对象，为后续以下能力做准备：

- 视角旋转
- 旋转中心变更
- 视口缩放
- 裁剪空间更新
- 鼠标交互扩展

本版本要求在不破坏当前渲染链路的前提下完成相机解耦，并保持控制面板现有接口可继续使用。

## 2. 本版本范围

### 2.1 包含内容

- 新增独立相机类
- 新增轨道相机控制器类
- 将 `Viewport` 中相机状态迁移到新类
- 保持 `Viewport -> FramePacket -> render_gl` 的现有数据流不变
- 保持当前控制面板中 `Distance`、`Vertical FOV` 的外部行为不变

### 2.2 不包含内容

- 鼠标旋转、滚轮缩放等交互实现
- 控制面板拆分为独立子 Widget
- `TextureHandle` 与纹理材质接入
- 正交相机支持
- 拾取、射线投射、视锥裁剪

## 3. 设计原则

### 3.1 相机职责拆分

建议分成两个类：

1. `Camera`
- 负责维护相机参数与输出 `scene_contract::CameraData`
- 管理 `position / target / up / fov / near / far / aspect`

2. `OrbitCameraController`
- 负责轨道相机控制逻辑
- 管理 `yaw / pitch / orbitCenter / distance`
- 对外提供后续交互扩展入口

### 3.2 模块边界

- `scene_contract` 继续只保留 `CameraData` 作为共享数据协议
- `test_viewer` 持有相机控制与交互逻辑
- `render_core` 与 `render_gl` 只消费 `CameraData`

## 4. 分步骤执行计划

### 步骤 1：建立独立相机类

目标：

- 在 `test_viewer` 内新增 `Camera` 类
- 将当前 `makeCamera(...)` 的矩阵构造能力迁移到 `Camera`
- 保持输出仍为 `scene_contract::CameraData`

预计改动：

- 新增相机头文件与实现文件
- 调整 `Viewport::rebuildFramePacket()` 的相机来源

完成判定：

- `Viewport` 不再依赖自由函数 `makeCamera(...)`
- 当前控制面板的相机参数仍可驱动视图变化

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 2

### 步骤 2：建立轨道相机控制器

目标：

- 新增 `OrbitCameraController`
- 将 `cameraDistance`、`verticalFov`、目标点等状态迁移到控制器
- 让 `Viewport` 通过控制器获取 `CameraData`

预计改动：

- `Viewport` 中移除直接相机参数存储
- 相机状态收口到控制器

完成判定：

- `Viewport` 仅负责调用控制器，不再自行组织相机数学状态
- 控制面板接口保持不变

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 3

### 步骤 3：补齐视口尺寸与裁剪参数更新链路

目标：

- 让控制器或相机对象明确维护 `aspect / near / far / fov`
- 在窗口尺寸变化时，更新投影参数
- 为后续鼠标交互和旋转中心扩展预留接口

预计改动：

- `resizeGL()` 或相关调用路径接入相机投影参数更新
- 增补必要的注释与命名清理

完成判定：

- 相机投影更新链路不再散落在 `Viewport` 逻辑中
- 相机模块具备后续扩展基础

测试与构建：

- 由用户执行
- 用户确认通过后本版本开发完成

## 5. 版本完成后的收尾要求
