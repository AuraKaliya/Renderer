# 0.0.16 开发记录

## 1. 版本目标

围绕“模型参数变化后的视角保持与自动适配策略”继续扩展 `test_viewer` 的视口行为，并将当前 `localBounds / worldBounds / sceneBounds / clip` 的基础链路进一步收口，使模型变化后的相机处理不再混同为自动 `Focus All`，而是形成：

- 保持当前视角
- 自动适配视图

两种明确策略的最小闭环。

## 2. 实际完成内容

### 2.1 建立模型变化策略的最小状态与调度入口

- 新增：
  - `src/test_viewer/src/model_change_view_state.h`
  - `src/test_viewer/src/model_change_view_state.cpp`
- 引入模型变化视角策略的最小状态结构，当前包含：
  - `ViewStrategy`
    - `keep_view`
    - `auto_frame`
  - `ChangeKind`
  - `PendingActions`
  - `pendingModelChange`
- 在 `Viewport` 中新增：
  - `modelChangeViewState_`
  - `setModelChangeViewStrategy(...)`
  - `modelChangeViewStrategy()`
  - `markModelChanged(...)`
  - `processPendingModelChange()`

### 2.2 接入模型变化后的 bounds / clip 更新闭环

- 在 `viewer_window.cpp` 中新增：
  - `mergeAabb(...)`
  - `visibleSceneWorldBounds()`
- 当前将“模型变化”的最小验证入口先落在：
  - 对象显隐切换 `setObjectVisible(...)`
- 当对象显隐变化时，系统会：
  1. 标记一次模型变化
  2. 更新当前场景 transform
  3. 聚合可见对象的世界空间 bounds
  4. 更新当前相机 `near/far`
- 在 `keep_view` 语义下，以上流程不会主动改写当前 framing。

### 2.3 接入自动适配视图策略的最小闭环

- `processPendingModelChange()` 现已根据策略分支：
  - `keep_view`
    - 仅更新 bounds 与 `near/far`
  - `auto_frame`
    - 调用 `applyFocusBounds(visibleFocusBounds())`
    - 自动将当前视图适配到可见对象
- 当前 `auto_frame` 与 `keep_view` 的差异，已经能通过对象显隐切换进行最小验证。

### 2.4 为测试界面补充最小验证入口

- 在 `ViewerControlPanel` 的 `Actions` 区新增：
  - `Model Change View`
- 提供两个最小选项：
  - `Keep View`
  - `Auto Frame`
- 面板状态通过 `PanelState.modelChangeViewStrategy` 与 `Viewport` 双向同步。

### 2.5 收口与 Focus / Viewport Zoom 的边界

- 本版本明确：
  - 模型变化策略独立于 `Focus`
  - 模型变化策略独立于 `Viewport Zoom`
- 以下行为继续保留其显式 framing 语义，不归入模型变化策略：
  - `Reset Defaults`
  - `Focus Sphere`
  - `Focus All`
  - `Focus Point`
- `Viewport Zoom` 继续保持 `0.0.15` 的独立交互闭环，不被本版本混入策略切换职责。

## 6. 本版本问题修复

- 本版本没有新增独立问题归档
- 主要修复与收口点为：
  - 将“模型变化后的相机处理”从原有 `Focus` 语义中拆出
  - 避免把模型变化直接等同于自动 `Focus All`
  - 明确 `keep_view` 与 `auto_frame` 的最小策略分层

## 7. 已知问题与限制

- 当前“模型变化”的最小验证入口仅覆盖：
  - 对象显隐切换
- 还未覆盖更完整的参数化模型重建或几何参数编辑路径
- 当前 `auto_frame` 复用现有 `applyFocusBounds(...)` 路径，仍属于第一版最小实现
- 当前尚未将 `sceneBounds` 作为更明确的公共状态对外暴露
- 本版本仍未扩展：
  - 选中系统
  - 旋转中心可视化
  - 更完整的 `virtual target / tumble pivot / workspace` 系统

## 8. 本版本遗留项

- 后续继续评审：
  - 是否需要把“模型变化”进一步细分为：
    - 场景显隐变化
    - 几何尺寸变化
    - 参数化重建
  - 是否需要为模型变化策略提供更正式的 UI 承载位置
  - 是否需要把 `sceneBounds` 提升为更明确的公共状态
  - 是否需要为 `auto_frame` 增加更多限制条件或策略选项
