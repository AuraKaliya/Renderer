# 0.0.11 开发计划
## 1. 版本目标

围绕正交模式下的缩放与聚焦行为，补齐与透视模式分离的 helper 分支和控制器行为，使正交模式具备更自然的默认导航语义，并为后续 `Lens Zoom` 与多组合缩放模式扩展打基础。

## 2. 本版本范围
### 2.1 包含内容

- 为正交模式新增独立的聚焦与缩放计算分支
- 明确正交模式默认主缩放为 `Lens`
- 让 `Focus All / Focus Bounds` 在正交模式下以 `orthographicHeight` 拟合为主
- 让 `Focus Point` 在正交模式下保持当前 `orthographicHeight`
- 保持透视模式当前行为不回归

### 2.2 不包含内容

- `Perspective + Lens` 的实际实现
- 投影模式与缩放模式的完整 UI 方案
- `render_core` / `render_gl` 协议层扩展
- 正交模式下完整交互重构

## 3. 已确认的设计约束

### 3.1 缩放语义约束

- `Dolly`
  - 改变相机真实距离 `distance`
  - 不主动改变投影视体参数
- `Lens`
  - 改变投影视体大小
  - 不改变相机真实位置

### 3.2 正交模式约束

- `Orthographic` 默认主缩放模式采用 `Lens`
- `orthographicHeight` 表示正交视体在垂直方向上的总高度
- 正交模式下：
  - `Focus All / Focus Bounds` 主要更新 `orthographicHeight`
  - `Focus Point` 仅更新 `orbitCenter`

### 3.3 透视模式约束

- 当前透视模式默认行为继续保持：
  - `Perspective`
  - `Dolly`
- 透视模式的 `Focus All / Focus Point / Focus Sphere` 行为不应回归

## 4. 分步骤执行计划
### 步骤 1：拆分正交模式 focus helper 分支

目标：

- 将当前透视导向的 `camera_focus_utils` 扩展为区分：
  - `Perspective`
  - `Orthographic`
- 为正交模式新增基于 `orthographicHeight` 的 framing 计算

完成判定：

- 正交模式下的 `Focus Bounds` 不再依赖 `FOV -> distance` 公式
- 工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 2

### 步骤 2：调整控制器缩放与聚焦行为

目标：

- `OrbitCameraController` 在正交模式下默认缩放采用 `Lens`
- `Focus All / Focus Bounds / Focus Point` 根据投影模式走不同分支
- `near/far` 更新逻辑保持可用

完成判定：

- 正交模式下滚轮缩放不再只改 `distance`
- 正交模式下聚焦行为符合已确认语义
- 工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 3

### 步骤 3：补最小验证入口并整理后续输入

目标：

完成判定：

- 用户可在当前测试界面验证正交模式缩放与聚焦行为
- 本版本结构与行为收口完成

测试与构建：

- 由用户执行
- 用户确认通过后本版本开发完成

## 5. 风险与注意事项

- 当前 `camera_clip_utils` 主要按透视模式经验值构造，本版本只要求保证正交模式下不明显失效，不扩展完整深度策略
- 不允许在本版本执行中插入 `Perspective + Lens`
- 若出现阻塞问题，按 `DebugDoc` 流程处理；若出现非阻塞疏漏，按 `ReviewDoc` 流程登记
