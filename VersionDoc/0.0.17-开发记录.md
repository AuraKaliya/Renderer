# 0.0.17 开发记录

## 1. 版本目标

围绕参数化建模的第一阶段能力建设，继续扩展 `parametric_model` 与 `test_viewer`，将当前“固定 primitive 直接生成 mesh”的路径升级为“基础 primitive 参数对象 + 可重建 mesh”的最小闭环，使项目正式具备：

- 参数化 primitive 对象
- 参数变化后的 mesh 重建
- 参数变化后的 bounds 更新
- 与 `0.0.16` 视图策略联动

的第一版能力。

## 2. 实际完成内容

### 2.1 建立基础 primitive 的显式参数描述与统一构建入口

- 在 `parametric_model` 中新增了基础参数协议：
  - `PrimitiveKind`
  - `BoxSpec`
  - `CylinderSpec`
  - `SphereSpec`
  - `PrimitiveDescriptor`
- 在 `PrimitiveFactory` 中新增：
  - `makeBoxDescriptor(...)`
  - `makeCylinderDescriptor(...)`
  - `makeSphereDescriptor(...)`
  - `build(const PrimitiveDescriptor&)`
- `build(...)` 当前负责：
  - 参数归一化
  - 最小合法值保护
  - 生成 `MeshData`
  - 输出 `localBounds`
- 旧接口：
  - `makeBox(...)`
  - `makeCylinder(...)`
  - `makeSphere(...)`
  继续保留，但已改为兼容封装到新的 descriptor/build 路径。

### 2.2 在 test_viewer 中接入 primitive 参数状态

- `Viewport::SceneObject` 现在不再只持有求值结果 `meshData`，还会持有：
  - `primitiveDescriptor`
- 三个默认对象现在都基于默认 descriptor 初始化：
  - Box
  - Cylinder
  - Sphere
- `Reset Defaults` 与 `Focus Sphere` 也会同步重建默认 descriptor，避免参数状态与几何状态脱节。

### 2.3 接入参数变化后的 mesh / bounds / GPU 重建闭环

- 在 `Viewport` 中新增：
  - `rebuildObjectMesh(int index)`
  - `applyPrimitiveDescriptor(int index, const PrimitiveDescriptor&)`
- 参数变化后的统一路径为：

`primitiveDescriptor 变化 -> PrimitiveFactory::build(...) -> meshData 更新 -> GPU mesh 更新/重上传 -> repository localBounds 更新 -> 标记模型变化 -> 交给 0.0.16 视图策略消费`

- 当前 GPU 路径优先使用：
  - `renderer_.updateMesh(...)`
- 若更新失败，则回退到：
  - `releaseMesh(...)`
  - `uploadMesh(...)`
  - `repository_.updateMeshHandle(...)`

### 2.4 提供三类 primitive 的最小参数编辑入口

- 在 `ViewerControlPanel` 中补齐了“构建整个模型所需最小参数量”的界面入口：

Box：
- `Width`
- `Height`
- `Depth`

Cylinder：
- `Radius`
- `Height`
- `Segments`

Sphere：
- `Radius`
- `Slices`
- `Stacks`

- 这些入口全部走统一 descriptor 重建路径，不再分散为临时特例。

### 2.5 与 0.0.16 视图策略联动

- 本版本明确将 primitive 参数变化归类为：
  - `model_change_view::ChangeKind::geometry_or_transform`
- 因此参数变化后会直接接入 `0.0.16` 的最小策略闭环：
  - `keep_view`
  - `auto_frame`
- 这意味着 primitive 参数编辑已经不只是几何变化，而是进入了统一的“模型变化 -> 视图处理”链路。

## 6. 本版本问题修复

- 在执行过程中，曾先用单一 `Box Width` 作为最小参数入口验证重建闭环
- 经讨论后，已明确：
  - 对所有参数化建模对象，都应提供“构建整个模型所需最小参数量”
- 因此本版本已将单一参数入口扩展为完整的：
  - Box 三轴参数
  - Cylinder 基础参数
  - Sphere 基础参数

## 7. 已知问题与限制

- 当前参数化系统仍然是“单体 primitive 参数对象化”阶段
- 当前一个对象仍然只有：
  - 一个基础 primitive descriptor
  - 一个求值结果 mesh
- 尚未引入：
  - 算子栈
  - 历史链
  - 曲线驱动生成
  - 布尔建模
  - 实例化
- 当前参数编辑入口仍集中在 `ViewerControlPanel` 的线性面板中，尚未形成更专业的对象参数面板或节点式承载

## 8. 本版本遗留项

- 后续继续评审：
  - 是否在 `parametric_model` 中引入更正式的对象级参数描述，而不是只停留在 primitive descriptor
  - 是否为参数化对象增加唯一类型与运行时识别结构
  - 是否将参数编辑入口从 `Actions` 区迁移到更合理的对象参数分组
  - 是否在下一版引入最小建模算子栈
