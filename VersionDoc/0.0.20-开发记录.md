# 0.0.20-开发记录
## 1. 版本目标

围绕参数化建模与选中系统的下一阶段演进，先将当前项目从“固定测试对象 + descriptor 拼装”的测试结构，推进到“正式参数化对象 + 显式特征链 + 最小选中闭环”的第一版。

本版本的目标是：

- 建立正式的参数化对象模型
- 为对象与特征引入稳定 id
- 让测试界面显式承载对象层与特征层
- 建立非拾取式的最小 selected / active 语义闭环

## 2. 实际完成内容

### 2.1 参数化对象结构正式对象化

- `ParametricObjectDescriptor` 从原先的：
  - `basePrimitive`
  - `operators`
  重构为：
  - `metadata`
  - `features`
- 新增：
  - `ParametricObjectId`
  - `ParametricFeatureId`
  - `FeatureKind`
  - `ParametricObjectMetadata`
  - `FeatureDescriptor`
- `PrimitiveFactory` 现在支持：
  - 创建 primitive feature
  - 创建 mirror feature
  - 创建 linear array feature
- `PrimitiveFactory::build(const ParametricObjectDescriptor&)` 改为按 feature 链顺序求值 mesh，而不是固定走“基础 primitive + operators”路径

### 2.2 Viewer 侧完成对新 feature 链结构的接线

- `ViewerWindow::Viewport` 不再直接依赖旧的 `basePrimitive / operators` 语义
- 新增并接入：
  - `ensureObjectFeature`
  - `findObjectFeature`
  - `findObjectFeatureById`
  - `ensureObjectPrimitive`
  - `findObjectPrimitive`
- 原有 box / cylinder / sphere 参数入口已经全部改为操作 primitive feature
- 原有 mirror / linear array 参数入口已经全部改为操作对应 feature
- 现有 mesh rebuild、bounds update、repository / GPU update 和 model change view 链路继续保持可用

### 2.3 测试界面显式加入对象层与特征层浏览

- `Scene` 页顶部新增 `Object Explorer`
- 新增对象列表，显式展示：
  - 对象类型
  - 对象 id
  - 当前对象 feature 数量
- 新增当前对象的 feature 列表，显式展示：
  - feature kind
  - feature id
  - enabled / disabled 状态
- 提供最小 feature 操作入口：
  - `Add Mirror Feature`
  - `Add Linear Array Feature`
  - `Remove Selected Feature`
  - `Selected Feature Enabled`

### 2.4 建立最小 SelectionState 与激活闭环

- `Viewport` 内新增最小 `SelectionState`，显式维护：
  - `selectedObjectId`
  - `activeObjectId`
  - `selectedFeatureId`
  - `activeFeatureId`
- 控制面板中新增 selected / active 状态展示
- 新增最小交互入口：
  - `Activate Selected Object`
  - `Activate Selected Feature`
  - `Focus Selected Object`
- 对象列表当前项现在驱动 `selected object`
- 特征列表当前项现在驱动 `selected feature`
- 激活按钮显式驱动 `active object / active feature`
- 删除 feature、重置默认场景或对象状态变化后，会通过 `normalizeSelectionState()` 做最小回退，避免 selected / active 悬空

### 2.5 保持本版本范围收口

- 本版本没有引入屏幕拾取
- 本版本没有引入控制点显示与拖拽
- 本版本没有引入 gizmo
- 本版本没有引入 raw mesh component 编辑
- 本版本仍以 `test_viewer` 作为最小验证入口，没有扩展成完整编辑器

## 6. 本版本问题修复

- 本版本没有新增需要单独归档到 `DebugDoc/` 的阻塞问题
- 开发过程中未形成需要单独记录的问题闭环文档

## 7. 已知问题与限制

- 当前对象列表仍是测试界面中的最小对象浏览区，不是完整对象树
- 当前 feature 列表仍为最小结构验证入口，尚未扩展出顺序调整、重命名等编辑能力
- 选中系统目前仍然是非拾取式的，仅由列表驱动
- 当前 active / selected 状态还没有与视口高亮、gizmo、控制点显示建立联动
- 当前对象依旧是固定 3 个测试对象，尚未进入真正的对象增删体系

## 8. 本版本遗留项

- 后续可继续评估：
  - 是否在下一版引入正式的对象增删入口
  - 是否让 feature 列表支持顺序调整与更明确的特征语义展示
  - 是否将当前 `SelectionState` 继续扩展到视口高亮与聚焦策略
  - 是否开始引入控制点描述与最小控制点显示逻辑

## 9. 下一版本建议

建议下一版本优先讨论并规划：

1. 正式对象增删与对象列表扩展
2. feature 顺序、命名和更清晰的特征编辑语义
3. 控制点描述结构与最小视口显示
4. selected / active 与视口高亮、局部编辑的联动方式
