# 0.0.1-开发记录

## 1. 版本目标

将 `test_viewer` 中的相机能力从 `Viewport` 内部状态中拆出，形成独立的相机对象与轨道相机控制器对象，并补齐视口尺寸与投影参数更新链路，为后续的视角旋转、旋转中心变更、视口缩放和裁剪空间更新提供基础。

## 2. 实际完成内容

### 2.1 相机能力拆分

- 新增独立 `Camera` 类
- 将相机矩阵构造从 `Viewport` 内自由函数迁移到 `Camera`

### 2.2 轨道相机控制器

- 新增 `OrbitCameraController`
- 将以下状态从 `Viewport` 迁移到控制器：
  - 观察中心
  - 距离
  - FOV
  - yaw / pitch
  - viewport size
  - near / far clip

### 2.3 视口与投影更新链路

- 在 `resizeGL()` 中同步视口尺寸到控制器
- 在 `initializeGL()` 首次构帧前同步当前尺寸
- 统一通过控制器输出 `CameraData`

### 2.4 问题修复

- 修复拖动改变窗口大小时，渲染区域出现控制面板残留的问题
- 修复 `Focus Sphere` 预设时球体未位于视图中心的问题
- 问题归档见：
  - `DebugDoc/0.0.1-问题.md`

## 5. 已知问题与限制

- 当前控制面板仍然全部堆叠在 `ViewerWindow::buildControlPanel()` 中，尚未拆分为独立 Widget
- 当前 `OrbitCameraController` 仅完成状态承载与投影链路收口，尚未接入鼠标交互
- 当前材质仍以纯色为主，尚未引入 `TextureHandle`

## 6. 下一版本建议

建议 `0.0.2` 聚焦于控制面板组件化拆分：

- 提取 `ViewerControlPanel`
- 提取 `SceneObjectControlWidget`
- 提取 `LightingControlWidget`
