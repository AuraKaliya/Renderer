# 0.0.3-开发计划

## 1. 版本目标

在不破坏模块独立性的前提下，引入 `TextureHandle` 与纹理资源协议，为材质从纯色扩展到纹理采样建立基础。

## 2. 本版本范围

### 2.1 包含内容

- 在 `scene_contract` 中定义纹理句柄和纹理数据协议
- 扩展 `MaterialData`，支持引用纹理句柄
- 在 `render_gl` 中增加纹理资源上传、更新、释放接口
- 为后续采样路径保留后端缓存入口

### 2.2 不包含内容

- 纹理文件加载器
- 图片解码库接入
- 复杂材质系统
- 法线贴图、PBR 等高级材质
- 控制面板中的纹理编辑 UI

## 3. 设计原则

### 3.1 模块独立性

- `scene_contract` 只定义纹理协议与资源句柄
- `render_gl` 负责 GPU 纹理对象创建与管理
- `parametric_model` 不依赖 OpenGL
- `test_viewer` 仅负责演示与调用

### 3.2 句柄边界

- 共享层只允许出现 `TextureHandle`
- 不允许把 `GLuint` 暴露到 `scene_contract`、`render_core` 或 `parametric_model`

## 4. 分步骤执行计划

### 步骤 1：建立共享纹理协议与后端接口

目标：

- 新增 `TextureHandle`
- 新增 `TextureData`
- 扩展 `MaterialData`
- 在 `render_gl::GlRenderer` 中增加纹理资源接口

完成判定：

- `scene_contract` 已具备纹理资源描述能力
- `render_gl` 已具备纹理资源管理接口
- 现有纯色材质路径不被破坏

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 2

### 步骤 2：接入 OpenGL 纹理缓存与材质采样路径

目标：

- 在 `render_gl` 中建立 `TextureHandle -> GL texture resource` 缓存
- 扩展 shader 和材质绑定逻辑
- 保持无纹理材质仍可正常工作

完成判定：

- 后端已能真正持有纹理 GPU 资源
- 纯色和纹理两条材质路径都存在

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 3

### 步骤 3：在 test_viewer 中接入最小纹理演示

目标：

- 在 `test_viewer` 中创建最小纹理资源
- 用示例物体验证纹理材质链路

完成判定：

- 测试界面可以展示纹理材质结果
- 当前资源缓存链路能够贯通

测试与构建：

- 由用户执行
- 用户确认通过后本版本开发完成
