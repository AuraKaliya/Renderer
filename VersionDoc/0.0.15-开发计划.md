# 0.0.15 开发计划

## 1. 版本目标

围绕 `Viewport Zoom` 作为“局部工作窗口缩放”的语义，继续扩展 `test_viewer` 的视口交互能力，使其更接近 Blender 一类 DCC 软件的工作方式：以鼠标位置作为缩放锚点，通过调整当前相机工作中心并复用现有缩放路径，建立一个缩放后仍可继续进行旋转、平移与后续缩放的可工作窗口。

## 2. 本版本范围

### 2.1 包含内容

- 重新定义 `Viewport Zoom` 的语义边界
- 为 `Viewport Zoom` 建立独立于普通 `Dolly / Lens` 的最小交互路径
- 在测试界面中接入 `Viewport Zoom` 的最小验证入口
- 让 `Viewport Zoom` 在第一版中支持：
  - 鼠标锚点
  - 工作中心调整
  - 缩放后继续 orbit / pan / zoom
- 保持当前 `Perspective + Dolly`、`Perspective + Lens`、`Orthographic + Lens` 行为不回归

### 2.2 不包含内容

- 模型参数变化后的：
  - 保持当前视角
  - 自动适配视图
  策略实现
- `Orthographic + Dolly` 的 UI 开放
- 旋转中心与选中系统扩展
- `render_core` / `render_gl` 协议层扩展
- 完整拾取系统
- 完整的视口偏移 / 工作空间重构系统

## 4. 已确认的设计约束

### 4.1 Viewport Zoom 语义约束

- `Viewport Zoom` 必须作为独立概念存在，不归类为普通 `Dolly` 或普通 `Lens`
- `Viewport Zoom` 不是纯屏幕空间放大，而是局部工作窗口缩放
- `Viewport Zoom` 会影响相机工作状态，当前优先考虑通过 `orbitCenter` 或等价工作中心体现
- `Viewport Zoom` 执行后，新的窗口必须仍能继续承接：
  - orbit
  - pan
  - zoom

### 4.2 当前阶段实现约束

- 第一版允许将 `Viewport Zoom` 作为特殊交互流程实现
- 第一版允许内部复用现有：
  - `Dolly`
  - `Lens`
  路径
- 但不允许将 `Viewport Zoom` 在语义上直接等同为普通 `Dolly` 或普通 `Lens`
- 第一版不引入完整拾取系统

### 4.3 鼠标锚点约束

- 当前优先以鼠标位置作为缩放锚点
- 第一版锚点优先采用：
  - 鼠标射线与经过当前 `orbitCenter` 的工作平面交点
- 若后续需要更高精度，再评审对象命中点或更复杂工作空间系统

  - 实际缩放路径

完成判定：
