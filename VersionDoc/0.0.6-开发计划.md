# 0.0.6-开发计划
## 1. 版本目标

在已建立 local bounds 和范围数据接入点的基础上，补齐 world bounds、scene bounds 与 `orbitCenter` 的显式状态接口，为后续 `Focus All / Focus Point` 以及基于范围的裁剪参数更新打基础。

## 2. 本版本范围
### 2.1 包含内容

- 在 `render_core` 中补充 world bounds 与 scene bounds 的基础组织能力
- 在测试界面中显式暴露 `orbitCenter` 只读状态
- 为后续 `Focus All / Focus Point` 预留最小接口

### 2.2 不包含内容

- 屏幕拾取
- 选中系统
- `Focus Selected`
- 基于 bounds 的自动 `near/far` 更新
- 视锥裁剪
- 调试绘制

## 3. 设计原则

### 3.1 范围优先

- 先把范围数据链路补齐，再引入更高阶交互行为
- 不把拾取和裁剪提前混入本版本

### 3.2 空间约定

- `local bounds` 继续保持模型空间
- `world bounds` 统一为世界空间
- `orbitCenter` 统一按世界空间定义

### 3.3 最小可用

- 第一阶段只做 AABB 链路
- 先保证数据可用、可观察、可扩展

## 4. 分步骤执行计划
### 步骤 1：补齐 world bounds 与 scene bounds 的基础数据接口

目标：
- 在 `render_core` 中增加 world bounds 数据组织能力
- 增加 scene bounds 聚合结果
- 保持当前渲染与场景装配路径不回归

完成判定：
- `SceneRepository` 或等价层已能提供对象世界范围数据
- 已能获得场景整体 bounds
- 当前工程可正常构建

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 2

### 步骤 2：显式化 orbitCenter 并补测试界面只读展示

目标：
- 在相机控制层明确 `orbitCenter` 的读写接口
- 在测试界面中增加 `orbitCenter` 的只读显示
- 保持现有交互行为不回归

完成判定：
- `orbitCenter` 已成为清晰的状态接口
- 控制面板中可观察当前 `orbitCenter`
- 当前交互行为正常

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 3

### 步骤 3：为 Focus All / Focus Point 预留最小接口

目标：
- 基于 scene bounds 和世界空间中心预留基础聚焦接口
- 暂不做完整交互动作，但建立后续扩展入口

完成判定：
- 已具备后续接入 `Focus All / Focus Point` 的基础接口
- 当前版本的数据链路完整闭环

测试与构建：

- 由用户执行
- 用户确认通过后本版本开发完成

## 5. 版本完成后的收尾要求

当本版本全部步骤完成后，需在 `VersionDoc` 下生成：

- `0.0.6-开发记录.md`

并按流程执行一次版本提交。

## 6. 当前版本执行规则

- 构建与测试由用户执行
- 每完成一个步骤后暂停等待用户验证
- 用户确认通过后再进入下一步骤
- 执行中如发现非阻塞疏漏，登记到 `ReviewDoc/`
