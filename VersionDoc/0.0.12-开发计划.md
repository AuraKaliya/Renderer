# 0.0.12 开发计划
## 1. 版本目标

围绕 `Perspective + Lens` 的行为边界，补齐透视模式下的可选镜头缩放路径，并保持当前默认透视导航、聚焦与裁剪语义稳定，为后续 `Zoom Mode` UI 策略评审提供可验证基础。

## 2. 本版本范围
### 2.1 包含内容

- 在透视模式下实现 `Lens Zoom` 的实际控制器行为
- 保持 `Perspective + Dolly` 作为默认透视主路径
- 在 `Perspective + Lens` 下：
  - 滚轮缩放只改 `verticalFovDegrees`
  - `Focus Bounds` 仍优先改 `distance`
  - `Focus Point` 不自动改 `FOV`
- 补最小验证入口与状态回填

### 2.2 不包含内容

- `Viewport Zoom`
- 模型参数变化后的：
  - 保持当前视角
  - 自动适配视图
  策略实现
- `Zoom Mode` 的完整 UI 开放
- `render_core` / `render_gl` 协议层扩展

## 3. 输入来源

本版本计划吸收以下评审输入：

- `ReviewDoc/0.0.11-评审遗留项.md`
- `ReviewDoc/0.0.12-评审遗留项.md`
- `TalkDoc/2026-03-12-Perspective+Lens行为边界-讨论.md`

本版本明确不吸收以下后续输入：

- `Viewport Zoom`
- 模型参数变化策略

## 4. 已确认的设计约束

### 4.1 Perspective + Lens 语义

- `Perspective + Lens` 只修改 `verticalFovDegrees`
- 不主动修改：
  - `distance`
  - `orbitCenter`

### 4.2 聚焦与裁剪约束

- `Focus Bounds` 在 `Perspective + Lens` 下仍优先改 `distance`
- `Focus Point` 在 `Perspective + Lens` 下不自动改 `FOV`
- `near/far` 仍主要依据真实距离与目标范围更新

### 4.3 默认行为约束

- 默认透视模式继续保持：
  - `Perspective + Dolly`
- `Perspective + Lens` 作为可选模式存在，不替换默认行为

## 5. 分步骤执行计划

目标：

- 在透视模式下为 `Lens` 建立独立缩放分支
- 保持现有透视 `Dolly` 与正交默认行为不回归

完成判定：

- 透视模式下已具备“只改 `FOV`”的 `Lens` 缩放行为
- 工程可正常编译

- 由用户执行
- 用户确认通过后进入步骤 2

- 用户可切到 `Perspective + Lens`
- 可验证 `FOV` 驱动的镜头缩放行为
- 工程可正常编译

测试与构建：
