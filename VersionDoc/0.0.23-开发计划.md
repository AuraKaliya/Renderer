# 0.0.23-开发计划

## 1. 版本目标

围绕 `0.0.22` 已经完成的渲染队列与场景接口收口，开始推进视口交互与参数化对象编辑的第一步联动。

本版本目标是：

- 让视口支持最小对象拾取，而不是只能通过右侧对象列表选择对象
- 让视口拾取结果复用现有 `selected / active` 状态闭环
- 保持当前 Orbit / Pan / Viewport Zoom 交互语义稳定
- 为后续视口高亮、gizmo、控制点显示与参数化局部编辑预留基础

## 2. 本版本范围

### 2.1 包含内容

- 为 `Viewport` 增加屏幕位置到世界空间 ray 的最小计算入口
- 基于当前对象 world bounds 做 AABB 拾取
- 左键短点击命中对象后更新：
  - `selected object`
  - `active object`
  - `selected feature`
  - `active feature`
- 对象列表和 feature 列表随视口拾取同步刷新
- 为左键旋转加入最小拖拽阈值，避免点击拾取误触发旋转

### 2.2 不包含内容

- 三角级精确拾取
- 多选、框选、Shift 追加选择
- 视口高亮轮廓
- gizmo
- 控制点显示与拖拽
- feature 顺序调整
- raw mesh component 选择

## 3. 已确认设计约束

- 拾取目标优先停留在对象层，不直接进入 mesh 顶点 / 边 / 面选择
- 拾取使用已有 scene object 与 `SelectionState`，不新增第二套选择状态
- 本版只使用 world AABB，后续如需精确选择再扩展到三角级或 ID buffer
- 左键短点击用于拾取，超过拖拽阈值后继续保持当前轨道旋转语义

## 4. 分步骤执行计划

### 步骤 1：接入最小对象拾取

目标：

- 视口左键短点击可选择当前鼠标下的可见对象

建议范围：

- 增加 viewport ray helper
- 增加 ray / AABB intersection
- 增加 `pickObjectAt(...)`
- 在左键释放时根据拖拽阈值触发拾取

完成判定：

- 点击对象可更新 selected / active object
- 点击对象后右侧列表可同步到当前对象
- 拖拽旋转不被短点击拾取逻辑破坏

测试与构建：

- 本地先验证相关目标可编译到当前环境允许的边界
- 完整 GUI 行为由用户在 Qt 链接环境正常后执行确认

## 5. 风险与注意事项

- 当前完整 `test_viewer` 构建仍受 Qt/MSVC 链接环境影响，可能只能验证到源文件编译阶段
- AABB 拾取会比三角拾取更粗，复杂模型或空洞区域可能出现选中包围盒内对象的情况
- 旋转点击阈值需要保持足够小，避免影响已有 orbit 手感

## 6. 版本完成后的收尾要求

当本版本完成后，需要在 `VersionDoc/` 下生成：

- `0.0.23-开发记录.md`

并按项目规则视用户确认情况决定是否提交。
