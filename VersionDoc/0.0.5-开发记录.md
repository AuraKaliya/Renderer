# 0.0.5-开发记录
## 1. 版本目标

围绕旋转中心扩展开发，先补齐范围数据基础能力，为后续聚焦、选中、自动裁剪和可见性判断提供统一的数据协议。

## 2. 实际完成内容

### 2.1 建立 bounds 协议与 local bounds 输出
- 在 `scene_contract` 中新增 `Aabb`
- 在 `MeshData` 中增加 `localBounds`
- 为 `Box / Cylinder / Sphere` 输出模型空间 local bounds

### 2.2 为 render_core 预留范围数据接入点
- 在 `render_core` 中新增 `ItemRangeData`
- `SceneRepository` 增加：
  - `updateLocalBounds`
  - `rangeData`
  - `snapshotRangeData`
- 内部存储从单纯 `RenderableItem` 扩展为带范围元数据的记录结构

### 2.3 增加测试界面的最小范围数据验证入口
- 在 `ViewerControlPanel` 中新增只读 `Bounds` 分组
- 当前可直接观察 `Box / Cylinder / Sphere` 的 local bounds
- `Viewport` 提供 `objectLocalBounds()` 只读接口

### 2.4 评审遗留项沉淀
- 将 `Aabb::valid` 语义与 helper 不完整的问题登记到 `ReviewDoc/0.0.5-评审遗留项.md`
- 按流程作为后续 bounds 扩展的评审输入

## 6. 已知问题与限制

- 当前仅有 local bounds，尚未引入 world bounds 和 scene bounds
- `render_core` 的范围数据接入点当前仅完成存储预留，尚未进入裁剪或聚焦逻辑
- 测试界面当前只做了 bounds 文本验证，尚未接入调试绘制

## 7. 本版本遗留项

- `Aabb::valid` 的语义体系和 helper 仍不完整
- 该项已登记到 `ReviewDoc/0.0.5-评审遗留项.md`
- 建议在下一次与 bounds 扩展相关的评审中优先讨论

## 8. 下一版本建议

建议下一版本优先聚焦以下内容：

- world bounds 与 scene bounds
- `orbitCenter` 的显式化与聚焦接口
- 基于范围数据的基础裁剪参数更新
