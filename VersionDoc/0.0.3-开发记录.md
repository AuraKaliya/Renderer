# 0.0.3-开发记录
## 1. 版本目标

在不破坏模块独立性的前提下，引入 `TextureHandle` 与纹理资源协议，为材质从纯色扩展到纹理采样建立基础。

## 2. 实际完成内容

### 2.1 建立共享纹理协议
- 在 `scene_contract` 中新增 `TextureHandle`
- 新增 `TextureFormat` 与 `TextureData`
- 扩展 `MaterialData`，支持引用基础色纹理

### 2.2 建立后端纹理缓存与采样链路
- 在 `render_gl::GlRenderer` 中新增纹理上传、更新、释放接口
- 建立 `TextureHandle -> OpenGL texture resource` 缓存
- 扩展 shader 与材质绑定逻辑，支持纯色和纹理两条材质路径并存

### 2.3 接入最小纹理演示
- 在 `test_viewer` 中新增程序生成的棋盘纹理
- 默认让 `box` 使用纹理材质
- 保持 `cylinder` 与 `sphere` 继续走纯色材质路径

- 用户已完成构建与测试
- `0.0.3` 当前开发计划中的步骤已确认通过
- 现有纯色材质路径未回归
- 最小纹理演示已验证可用

## 5. 已知问题与限制

- 当前纹理仍为程序生成示例纹理，尚未接入文件加载与图片解码流程
- 控制面板尚未提供纹理切换或纹理参数编辑能力
- 视口相机仍主要依赖参数面板，鼠标交互尚未接入

## 6. 下一版本建议

建议 `0.0.4` 聚焦视口交互相机：
- 扩展 `OrbitCameraController` 的交互动作能力
- 在 `Viewport` 中接入鼠标旋转、缩放、平移
- 保持控制面板与视口交互状态同步
