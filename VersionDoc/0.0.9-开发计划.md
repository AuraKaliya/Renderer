# 0.0.9 开发计划

## 1. 版本目标

围绕当前相机与聚焦系统的代码结构进行整理，显式拆分 `render bounds` 与 `focus bounds` 语义，抽离聚焦范围计算 helper，并收紧 `Viewport` 内部职责，为后续正交模式与 `Lens Zoom` 接入打基础。

## 2. 本版本范围

### 2.1 包含内容

- 显式整理：
  - `render bounds`
  - `focus bounds`
  的语义边界
- 将当前 `Viewport` 内的稳定聚焦范围计算下沉到独立 helper
- 收紧 `Viewport` 中与聚焦范围相关的职责
- 保持当前：
  - `Focus All`
  - `Focus Point`
  - `Focus Sphere`
  - Dolly 缩放裁剪联动
  行为不回归

### 2.2 不包含内容

- 正交模式真实投影矩阵接入
- 正交模式 UI
- `Lens Zoom` 的行为实现
- 视锥裁剪
- 拾取与选中
- `Focus Selected`

## 3. 已确认的设计约束

### 3.1 Bounds 语义

- `render bounds`
  - 面向真实渲染与瞬时范围数据
- `focus bounds`
  - 面向聚焦行为与稳定范围数据
- 两者应显式区分，不继续隐式混用

### 3.2 模块职责

- `render_core`
  - 继续负责场景范围与装配范围数据
- `test_viewer`
  - 允许保留测试界面特有的聚焦语义
- 聚焦范围计算应由独立 helper 承担，不继续堆在 `Viewport`

### 3.3 相机工具层

- `camera_focus_utils`
  - 负责聚焦与 framing
- `camera_clip_utils`
  - 负责裁剪计算
- 本版本可新增：
  - `camera_focus_bounds_utils`
  或等价 helper

## 4. 分步骤执行计划

### 步骤 1：显式整理 bounds 语义与 helper 边界

目标：

- 明确当前代码中的：
  - `render bounds`
  - `focus bounds`
- 新增独立聚焦范围 helper
- 将稳定聚焦范围计算从 `Viewport` 中迁出

完成判定：

- `Viewport` 不再直接承载主要聚焦范围计算细节
- bounds 语义命名清晰
- 当前工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 2

### 步骤 2：收紧 Viewport 内部职责并复用新 helper

目标：

- `Focus All / Focus Sphere` 改为统一走新的聚焦范围 helper
- 保持滚轮缩放和聚焦链路仍能复用稳定范围数据
- 降低 `Viewport` 内部几何计算耦合

完成判定：

- `Viewport` 仅保留调度与状态协作职责
- 聚焦行为不回归
- 当前工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 3

### 步骤 3：整理文档与后续正交模式输入

目标：

- 为下一阶段“正交模式真实投影路径接入”整理明确输入

完成判定：

- 当前结构整理闭环完成
- 后续正交模式具备清晰输入边界
- 当前工程可正常编译

测试与构建：

## 5. 风险与注意事项

- 本版本不应把正交模式实现提前混入
- 若在整理过程中发现 `render_core` 与 `test_viewer` 的 bounds 语义仍有冲突，应优先保证当前行为稳定，再登记到 `ReviewDoc/`
- 结构整理后，应重点关注 `Focus All / Focus Sphere` 是否仍保持稳定
