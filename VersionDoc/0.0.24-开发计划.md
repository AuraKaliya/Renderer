# 0.0.24-开发计划

## 1. 版本目标

本版本围绕测试展示 UI 的后续演进，先落地 UI 适配层与接口打印测试基础。

目标是：

- 在 `test_viewer` 中新增一层 UI state adapter，降低 `Viewport` 与控制面板状态结构的直接耦合
- 为后续按经典编辑器布局拆分 `Scene / Viewport / Inspector / Debug` 功能区提供稳定数据入口
- 先保持当前 UI 行为不变，避免布局重构与状态适配同时改变导致验证困难
- 为适配层接口增加以打印和值确认为主的 smoke 测试

## 2. 本版本范围

### 2.1 包含内容

- 新增 Qt 无关的 UI 状态类型定义
- 新增 `ViewerUiStateAdapter`，负责把参数化对象、特征、节点、求值结果转换为 UI 可消费状态
- 将 `ViewerWindow::Viewport::controlPanelState()` 中的对象状态组装逻辑迁移到适配层
- 新增 adapter smoke test 可执行目标
- smoke test 采用打印与值确认方式覆盖：
  - 默认 Box 对象 UI 状态
  - feature / unit / input / node usage 描述
  - evaluation summary 与 diagnostic 计数

### 2.2 不包含内容

- 暂不重排主窗口布局
- 暂不拆分 `SceneHierarchyWidget / InspectorPanelWidget / DebugPanelWidget`
- 暂不引入完整 command 系统
- 暂不新增模型类型
- 暂不修改视口交互语义
- 暂不引入 undo / redo、gizmo、控制点拖拽或多选

## 4. 分步骤执行计划

### 步骤 1：新增 UI 状态适配层与 smoke 测试

目标：

- 先把对象状态、参数化结构状态、求值摘要从 `Viewport` 中迁移到独立 adapter
- 当前 UI 表现保持不变

建议范围：

- 新增 `viewer_ui_types.h`
- 新增 `viewer_ui_state_adapter.h/.cpp`
- 更新 `src/test_viewer/CMakeLists.txt`
- 更新 `ViewerWindow::Viewport::controlPanelState()`
- 新增 `viewer_ui_state_adapter_smoke_tests`

完成判定：

- 项目可构建
- smoke test 可运行并打印 `[PASS]`
- 当前控制面板仍能获取对象、feature、unit、diagnostic 等状态

### 步骤 2：按功能区拆分控制面板承载

目标：

- 将当前右侧分页面板逐步拆成经典编辑器式功能区的组件入口

建议范围：

- 新增或整理：
  - Scene/Hierarchy 区
  - Inspector 区
  - Debug 区
  - Toolbar/Overlay 区
- 继续复用步骤 1 的 UI state adapter

完成判定：

- 对象、feature、node 选择闭环不变
- primitive 参数编辑不变
- Debug 信息刷新不变

### 步骤 3：参数化参数区 schema 化

目标：

- 让 primitive / feature 参数编辑逐步由 construction schema 驱动

建议范围：

- 新增 `ParametricUiSchemaAdapter`
- 将 `float / integer / vector3 / enum / node` 输入抽象为 UI 字段
- 保留 Box/Cylinder/Sphere 现有行为作为验证基准

完成判定：

- Box/Cylinder/Sphere 的构造模式和参数编辑仍可用
- 参数区生成逻辑不再完全依赖三套硬编码控件

### 步骤 4：命令入口收口

目标：

- 将 UI 操作逐步收敛为统一 command 或 patch 接口

建议范围：

- 新增 `ViewerCommand` 轻量结构
- 新增 `applyViewerCommand(...)`
- 将参数化相关信号优先迁移到 command 入口

完成判定：

- 常用参数修改、feature 启停、node 坐标修改均能通过统一入口打印并确认结果

## 5. 测试与构建要求

- 每个新增适配接口至少有一组打印/值确认测试
- smoke test 失败时返回非 0
- 本地执行：
  - `cmake --build build --config Debug`
  - 对应 smoke test 可执行文件
- GUI 交互行为由用户继续进行人工验收

## 6. 风险与注意事项

- 当前 `ViewerControlPanel::PanelState` 仍然较大，步骤 1 不直接删除它，只先迁移状态组装来源
- 布局拆分要避免同时改变信号协议，否则回归点会过多
- schema 化参数区要避免提前绑定新模型类型，本版本优先服务重构基础
- 项目文档中存在 `0.0.23` 引用 `0.0.22` 但未见 `0.0.22` 计划/记录的情况；本版本先不修正历史文档，仅在后续收口时视需要处理
