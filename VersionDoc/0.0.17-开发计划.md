# 0.0.17 开发计划

## 1. 版本目标

围绕参数化建模的第一阶段能力建设，继续扩展 `parametric_model` 与 `test_viewer`，将当前“固定 primitive 直接生成 mesh”的路径升级为“基础 primitive 参数对象 + 可重建 mesh”的最小闭环，使项目正式具备：

- 参数化 primitive 对象
- 参数变化后的 mesh 重建
- 参数变化后的 bounds 更新
- 与 `0.0.16` 视图策略联动

的第一版能力。

## 2. 本版本范围

### 2.1 包含内容

- 为基础 primitive 建立显式参数描述
- 为基础 primitive 建立统一重建入口
- 在 `test_viewer` 中提供最小参数编辑入口
- 将参数变化接入：
  - mesh 重建
  - local bounds 更新
  - repository / GPU 资源更新
  - `keep_view / auto_frame` 视图策略
- 继续保持 `parametric_model` 与 OpenGL / Qt 解耦

### 2.2 不包含内容

- 完整建模算子栈
- 布尔建模
- 节点图系统
- 曲线/剖面驱动建模
- 选中系统与 gizmo
- 资源序列化 / 文件保存格式

## 3. 输入来源

本版本计划吸收以下输入：

- `TalkDoc/2026-04-02-参数化建模路线参考Blender与Maya-讨论.md`

## 4. 已确认的设计约束

### 4.1 参数化建模约束

- `parametric_model` 仍然只负责 CPU 侧几何生成
- 参数对象与 mesh 求值必须分层
- `MeshData` 仍然是当前阶段的求值输出，不直接承担“参数源数据”职责
- 参数变化后的重建必须走统一路径，不允许零散地手工更新：
  - mesh
  - bounds
  - GPU 资源

### 4.2 与现有渲染结构的边界

- 不允许把 OpenGL 资源句柄暴露进 `parametric_model`
- 不允许让 `render_gl` 感知 primitive 参数结构
- `test_viewer` 可以持有当前对象的参数状态，但求值逻辑仍由 `parametric_model` 负责

### 4.3 与视图策略的边界

- 参数变化属于“模型变化”
- 参数变化后的相机处理必须接入 `0.0.16` 已建立的：
  - `keep_view`
  - `auto_frame`
- 不允许把参数变化直接等同于手工 `Focus All`

## 5. 分步骤执行计划

### 步骤 1：建立基础 primitive 参数描述与统一构建入口

目标：

- 把当前 `makeBox / makeCylinder / makeSphere` 的直接调用，升级为“参数描述驱动的统一构建入口”

建议范围：

- 新增基础参数结构，例如：
  - `BoxSpec`
  - `CylinderSpec`
  - `SphereSpec`
- 建立统一 descriptor / build request
- 统一参数校验与默认值策略

完成判定：

- 基础 primitive 已有显式参数结构
- `parametric_model` 已存在统一构建入口
- 现有 fixed primitive 行为不回归

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 2

### 步骤 2：接入参数变化后的 mesh / bounds 重建闭环

目标：

- 当 primitive 参数变化时，系统能重建 mesh，并同步更新 bounds 与 GPU 资源

建议范围：

- 在 `test_viewer` 中为至少一个 primitive 接入参数变化入口
- 参数变化后自动完成：
  - 新 `MeshData` 求值
  - local bounds 更新
  - repository bounds 更新
  - GPU mesh 更新或重上传
- 将参数变化标记为模型变化，接入 `0.0.16` 策略

完成判定：

- 参数变化可以真实改变几何形状
- 参数变化后的 bounds 与裁剪范围正确
- `keep_view / auto_frame` 对参数变化已生效

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 3

### 步骤 3：扩展基础参数编辑入口并收口版本边界

目标：

- 在 `test_viewer` 中形成基础 primitive 参数编辑的最小验证界面
- 收口本版本只做“单体 primitive 参数对象化”，不扩展到算子栈

建议范围：

- 至少覆盖：
  - Box 的宽高深
  - Cylinder 的半径、高度、分段数
  - Sphere 的半径、切片/层数
- 明确参数合法范围
- 补充本版本开发记录

完成判定：

- 三类 primitive 均可进行最小参数编辑
- 参数变化 -> 重建 -> bounds / 视图策略闭环完整
- 本版本开发记录可清晰说明后续算子栈留给下一版

测试与构建：

- 由用户执行
- 用户确认通过后本版本开发完成

## 6. 风险与注意事项

- 不允许在本版本中插入完整算子栈
- 不允许在本版本中插入节点系统
- 若 GPU mesh 更新路径复杂度超出当前步范围，应先用最小可行重上传路径打通闭环
- 若出现当前步骤阻塞问题，按 `DebugDoc` 流程处理
- 若发现非阻塞优化点，按 `ReviewDoc` 流程登记

## 7. 版本完成后的收尾要求
