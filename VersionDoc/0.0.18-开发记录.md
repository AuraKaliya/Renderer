# 0.0.18-开发记录
## 1. 版本目标

围绕 `0.0.17` 已建立的基础 primitive 参数对象化能力，继续扩展 `parametric_model` 与 `test_viewer`，将当前“单一 primitive 直接求值 mesh”的结构升级为“基础 primitive + 最小建模算子栈”的第一版闭环，使项目正式具备：

- 对象级参数描述
- 固定顺序的最小算子求值
- 算子变化后的 mesh 重建
- 算子变化后的 bounds 更新
- 与 `0.0.16` 视图策略联动

的第一版能力。

## 2. 实际完成内容

### 2.1 建立对象级 descriptor 与最小算子协议

- 在 `parametric_model` 中新增了对象级描述结构：
  - `ParametricObjectDescriptor`
  - `OperatorDescriptor`
- 新增了首批算子协议：
  - `OperatorKind`
    - `mirror`
    - `linear_array`
  - `Axis`
  - `MirrorOperatorSpec`
  - `LinearArrayOperatorSpec`
- 在 `PrimitiveFactory` 中新增：
  - `makeParametricObject(...)`
  - `build(const ParametricObjectDescriptor&)`

当前对象级求值路径已经从：

`PrimitiveDescriptor -> MeshData`

升级为：

`ParametricObjectDescriptor -> MeshData`

### 2.2 实现 Mirror / Linear Array 的最小求值闭环

- 在 `parametric_model` 中实现了：
  - `applyMirror(...)`
  - `applyLinearArray(...)`
- 当前求值顺序固定为：
  1. 先求值 `basePrimitive`
  2. 再按 `operators` 顺序依次应用算子
  3. 输出最终 `MeshData`
  4. 重新计算 `localBounds`

- `Mirror` 当前支持：
  - `Enabled`
  - `Axis`
  - `planeOffset`
- `Linear Array` 当前支持：
  - `Enabled`
  - `Count`
  - `Offset X`
  - `Offset Y`
  - `Offset Z`

### 2.3 将对象级 descriptor 接入 test_viewer 的重建链路

- `Viewport::SceneObject` 不再只持有单一 primitive 参数，而是改为持有：
  - `parametricObjectDescriptor`
- 默认对象现在都通过对象级 descriptor 初始化，并统一走：
  - `PrimitiveFactory::build(const ParametricObjectDescriptor&)`
- 对象重建路径统一为：

`descriptor 变化 -> build(...) -> meshData 更新 -> GPU mesh 更新/重传 -> repository localBounds 更新 -> 标记模型变化 -> 交给 0.0.16 视图策略消费`

这意味着 `Mirror` 与 `Linear Array` 的变化已经正式进入：

- mesh 重建
- bounds 更新
- repository / GPU 更新
- `keep_view / auto_frame`

的统一链路。

### 2.4 在 test_viewer 中补最小 operator UI

- 在 `SceneObjectControlWidget` 中为每个固定对象补充了最小算子编辑入口：

Mirror：
- `Enabled`
- `Axis`
- `Mirror Offset`

Linear Array：
- `Enabled`
- `Count`
- `Offset X`
- `Offset Y`
- `Offset Z`

- 当前 UI 仍然采用固定顺序、固定对象的方式：
  - Box
  - Cylinder
  - Sphere

不引入：
- 对象选择系统
- 栈顺序拖拽
- 复杂 Modifier Stack 编辑器

### 2.5 Mirror 重叠问题修正为“带偏移平面”的最小语义

- 在执行过程中，发现当前 `Mirror` 若直接围绕局部原点镜像，完整且以原点为中心的 primitive 会与原模型完全重叠。
- 该表现对于当前最小数学实现并不算错误，但对建模算子体验不够实用。
- 因此本版追加收口：
  - 为 `MirrorOperatorSpec` 新增 `planeOffset`
  - 将位置镜像改为围绕可偏移平面执行
  - 法线镜像继续保持纯轴向翻转，不受位置偏移影响

当前 `Mirror` 已从“只能绕局部原点重叠镜像”升级为“绕可偏移平面镜像”的最小可用版本。

### 2.6 与 0.0.16 视图策略联动

- 本版明确将 operator 参数变化归类为：
  - `model_change_view::ChangeKind::geometry_or_transform`
- 因此算子变化后会继续接入 `0.0.16` 的最小策略闭环：
  - `keep_view`
  - `auto_frame`

这意味着当前算子编辑已经不只是几何实验，而是进入了统一的“模型变化 -> 视图处理”链路。

## 6. 本版本问题修复

- 在执行过程中，曾先实现“绕局部原点镜像”的最小 `Mirror`
- 经验证后发现：
  - 对完整且以原点为中心的 primitive，镜像结果会与原模型重叠
- 因此本版已进一步修正为：
  - 支持 `Mirror Offset`
  - 支持围绕可偏移平面镜像

该修正属于本版对最小算子语义的重要收口。

## 7. 已知问题与限制

- 当前算子系统仍是“固定顺序、固定对象、最小参数面板”的第一版
- 当前 `Mirror` 仅支持：
  - 平面轴向选择
  - 平面偏移

尚未支持：
- `bisect`
- `clip`
- `merge`
- 顶点焊接或 seam 处理

- 当前 `Linear Array` 仅支持线性重复与平移偏移，尚未支持：
  - 旋转阵列
  - 局部实例化
  - 中间缓存或增量求值

- 当前 UI 仍集中在固定对象面板中，尚未形成更正式的对象级建模面板或历史栈承载

## 8. 本版本遗留项
