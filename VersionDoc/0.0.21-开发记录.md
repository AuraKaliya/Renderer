# 0.0.21-开发记录

## 1. 版本目标

围绕 `0.0.20` 已建立的参数化对象、特征链与最小选中闭环，继续推进到“正式对象增删与动态对象列表”的第一版。
本版本的目标是：

- 将当前固定 3 个测试对象的承载方式推进为动态对象集合
- 为测试界面补齐最小对象增删入口
- 让对象列表真正成为对象层主入口，而不是固定 3 个对象的只读映射
- 保持现有 feature 列表与最小 `selected / active` 语义继续可用
- 让对象增删与 `focus / reset / preset` 形成稳定闭环

## 2. 实际完成内容

### 2.1 对象承载从固定三对象升级为动态集合

- `Viewport` 内部对象承载已从固定三对象结构切换为动态 `std::vector`
- bounds、focus、frame packet、selection 基础遍历逻辑已同步改为按动态对象集合工作
- 默认场景仍保持 3 个对象启动，但对象层核心逻辑不再依赖固定数量假设

### 2.2 测试界面接入正式对象增删入口

- `Scene` 页中的 `Object Explorer` 已新增：
  - `Add Box`
  - `Add Cylinder`
  - `Add Sphere`
  - `Delete Selected Object`
- 对象列表现在承载真实动态对象数量，而不是固定三对象映射
- 当前对象详情区已改为“对象列表驱动的当前对象检查器”
- 会根据当前对象 primitive 类型显示对应的 Box / Cylinder / Sphere 参数面板

### 2.3 当前对象检查器与对象列表正式对齐

- `ViewerControlPanel::PanelState` 已改为按对象携带 primitive 数据，而不是继续依赖顶层固定 `box / cylinder / sphere` 状态
- 当前对象检查器会回填：
  - 可见性
  - 旋转速度
  - 颜色
  - primitive 参数
  - mirror / linear array 状态
- bounds 区也已改为动态对象列表展示，而不再只显示固定三对象 bounds

### 2.4 Viewport 支持正式新增和删除对象

- 新增 `addObject(...)`
- 新增 `removeSelectedObject()`
- 新增对象时会：
  - 创建默认参数化对象 descriptor
  - 生成默认 mesh
  - 分配默认颜色、变换、可见性和旋转速度
  - 在渲染已初始化时上传 GPU 资源
  - 重建 repository item
- 删除对象时会：
  - 释放对应 mesh / material / texture 资源
  - 从动态对象集合中移除对象
  - 重建 repository item 索引

### 2.5 对象增删后的状态闭环已明确化

- 新增对象后，会直接成为当前：
  - `selected object`
  - `active object`
  - `selected feature`
  - `active feature`
- 删除当前对象后，会按相邻对象回退当前：
  - `selected object`
  - `active object`
  - `selected feature`
  - `active feature`
- 当对象集合为空时，`SelectionState` 会显式回退为全空状态，避免悬空 id

### 2.6 Reset / Focus 相关行为与动态对象集合对齐

- `Reset Defaults` 现在会完整重建默认对象集，而不是假定原对象永远存在
- `Focus Sphere` 现在会在恢复默认场景后，显式将球体对象设为当前 `selected + active`
- `Focus Selected Object`、`Focus All` 与对象增删后的对象集合继续共享统一 bounds / focus 路径

## 7. 已知问题与限制

- 当前对象列表仍是测试界面中的最小对象浏览区，不是完整对象树
- 当前对象检查器仍是单对象测试型详情区，尚未扩展为完整对象属性编辑器
- `SceneRepository` 目前没有单对象删除接口，本版删除对象通过重建 repository item 索引完成
- 当前选中系统依然是非拾取式的，主要由对象列表与 feature 列表驱动
- 当前 active / selected 状态尚未与视口高亮、gizmo、控制点显示建立联动

## 8. 本版本遗留项

- 后续可继续评估：
  - 是否为对象列表增加更明确的排序、命名或对象元数据展示
  - 是否为 feature 列表增加更明确的编辑语义与顺序管理
  - 是否把当前 `SelectionState` 继续扩展到视口高亮与控制点显示
  - 是否为对象创建引入更正式的默认布局或对象放置策略

## 9. 下一版本建议

建议下一版本优先讨论并规划：

1. feature 语义继续收口，包括顺序、命名和更清晰的特征层编辑承载
2. 控制点描述结构与最小视口显示逻辑
3. selected / active 与视口高亮、局部编辑入口的联动方式
