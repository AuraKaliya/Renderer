# 0.0.6-开发记录
## 1. 版本目标

在已建立 local bounds 和范围数据接入点的基础上，补齐 world bounds、scene bounds 与 `orbitCenter` 的显式状态接口，为后续 `Focus All / Focus Point` 以及基于范围的裁剪参数更新打基础。

## 2. 实际完成内容

### 2.1 补齐 world bounds 与 scene bounds 的基础数据接口
- 在 `render_core` 中为 `ItemRangeData` 增加 `worldBounds`
- 新增 `SceneRangeData`
- `SceneRepository` 已具备对象世界范围和场景整体范围的读取能力

### 2.2 显式化 orbitCenter 并补测试界面只读展示
- `Viewport` 显式提供 `setOrbitCenter()` 与 `orbitCenter()`
- `CameraControlWidget` 新增只读 `Orbit Center` 显示
- `ViewerControlPanel` 已能同步当前视口的 `orbitCenter`

### 2.3 为 Focus All / Focus Point 预留最小接口
- `OrbitCameraController` 新增：
  - `focusOnPoint()`
  - `focusOnBounds()`
- `Viewport` 新增：
  - `focusOnPoint()`
  - `focusOnScene()`
- 当前聚焦语义统一为世界空间目标点或范围中心

### 2.4 讨论记录与流程补充
- 补充 `TalkDoc/` 讨论留档规则
- 归档本次关于 `Orbit Center` 与相机朝向关系的讨论结果

## 4. 用户构建/测试结论

- 用户已完成构建与测试
- `0.0.6` 当前开发计划中的步骤已确认通过
- world bounds、scene bounds、orbitCenter 只读展示和最小聚焦接口均已可用
- 现有渲染、交互与控制面板能力未回归

## 5. 本版本问题修复

- 本版本未出现需要单独归档到 `DebugDoc/` 的阻塞问题修复项

## 6. 已知问题与限制

- `Focus All / Focus Point` 当前仅预留基础接口，尚未接入实际 UI 动作
- 当前聚焦接口未自动调整距离、FOV 或 `near/far`
- world bounds 和 scene bounds 目前只用于数据组织，尚未参与裁剪或聚焦策略

## 7. 本版本遗留项

- `Aabb::valid` 的语义体系和 helper 仍不完整
- 该项仍以 `ReviewDoc/0.0.5-评审遗留项.md` 作为后续输入
- 自动 `near/far` 更新、`Focus Selected`、拾取与调试绘制仍待后续评审拆解

## 8. 下一版本建议

建议下一版本优先聚焦以下内容：

- 将 `Focus All / Focus Point` 接入测试界面实际操作
- 基于范围数据补基础距离与裁剪参数更新
- 继续评审是否进入 `Focus Selected` 与拾取链路
