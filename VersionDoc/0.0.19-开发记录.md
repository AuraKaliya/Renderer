# 0.0.19-开发记录

## 1. 版本目标

围绕当前 `test_viewer` 测试/展示界面的信息架构问题，重构右侧控制面板的组织方式，使界面从“单一线性滚动面板”升级为“按分类组织的分组界面”，并将对象参数入口迁移到更符合对象编辑语义的位置。

本版本目标是：

- 为 `ViewerControlPanel` 建立顶层分类结构
- 将 Box / Cylinder / Sphere 的 primitive 参数迁移回对应对象面板
- 收口相机动作、调试信息与重置入口的职责边界
- 保持现有 `Viewport` 联动与状态回填能力不回归

## 2. 实际完成内容

### 2.1 新增测试界面分组与分类重构讨论输入

- 新增讨论文档：
  - `TalkDoc/2026-04-03-测试界面分组与分类重构-讨论.md`
- 明确本次版本采用：
  - 顶层分类
  - 分类内局部分组
  的重构方式
- 明确本版不扩展：
  - 对象选择系统
  - 节点式建模 UI
  - 完整 binder / adapter 层

### 2.2 将 `ViewerControlPanel` 从线性面板重构为顶层分类页

- `ViewerControlPanel` 不再把全部控件堆叠在同一条滚动流中
- 右侧控制面板现已重构为三个顶层分类页：
  - `Scene`
  - `Camera`
  - `Debug`
- 每个分类页内部继续保持独立滚动承载，避免回退到固定高度面板问题

当前顶层组织为：

- `Scene`
  - Box
  - Cylinder
  - Sphere
  - Lighting
- `Camera`
  - Camera
  - View Actions
- `Debug`
  - Bounds
  - Debug Actions

### 2.3 将 primitive 参数入口迁移回对应对象面板

- `SceneObjectControlWidget` 由原先的单层表单，重构为对象卡片内部的局部分组：
  - `Object`
  - `Appearance`
  - `Primitive`
  - `Operators`
- Box 参数迁回 Box 面板：
  - `Width`
  - `Height`
  - `Depth`
- Cylinder 参数迁回 Cylinder 面板：
  - `Radius`
  - `Height`
  - `Segments`
- Sphere 参数迁回 Sphere 面板：
  - `Radius`
  - `Slices`
  - `Stacks`

本次迁移后，对象的基础建模参数、外观参数与算子参数已经归位到同一对象卡片中，不再依赖全局 `Actions` 区编辑。

### 2.4 收口相机动作区与调试区职责边界

- 移除原先语义过于泛化的 `Actions` 统一分组
- 在 `Camera` 页新增 `View Actions`：
  - `Model Change View`
  - `Focus Sphere`
  - `Focus All`
- 在 `Debug` 页新增 `Debug Actions`：
  - `Reset Defaults`
- `Bounds` 继续保留在 `Debug` 页，作为只读调试信息展示区

这样本版本完成后，界面职责边界已经形成基本收口：

- 对象编辑留在 `Scene`
- 相机与视图动作留在 `Camera`
- 调试信息与重置入口留在 `Debug`

### 2.5 保持现有外部联动接口基本稳定

- 本版本没有把 UI 重组扩展成大规模接口重写
- `ViewerControlPanel` 仍然作为总装层存在
- `ViewerWindow -> ViewerControlPanel` 的主要信号/槽接口继续沿用
- `Viewport -> Panel` 的整体状态回填路径保持可用

本版重点调整的是承载结构和信息架构，而不是重写渲染、相机或建模联动逻辑。

## 7. 已知问题与限制

- 当前 `ViewerControlPanel` 虽然完成了顶层分类与对象参数归位，但仍由总装层直接承担较多信号转发职责
- `ViewerWindow` 仍直接承担控制面板信号绑定，尚未引入更明确的 binder / adapter 层
- 当前对象面板仍是固定对象卡片结构，尚未扩展到对象选择、多对象编辑或更正式的对象参数系统
- 当前界面重组主要服务于测试/验证用途，尚未进入更正式的编辑器级承载方式

## 8. 本版本遗留项

- 后续可继续评审：
  - 是否把 `Scene` / `Camera` / `Debug` 页进一步抽成独立 section widget
  - 是否继续收口 `ViewerWindow` 中的面板信号绑定职责
  - 是否为对象卡片增加更明确的状态展示或折叠策略
  - 是否在后续版本引入更正式的对象级参数面板承载

## 9. 下一版本建议

建议下一版本优先讨论：

1. 是否为 `ViewerWindow` 引入更明确的 `ViewerControlBinder` / `ViewerUiStateAdapter`
2. 是否将当前页内分组继续细化为更独立的 section widget
3. 是否围绕对象面板继续增强参数展示、调试状态与对象级信息组织方式
