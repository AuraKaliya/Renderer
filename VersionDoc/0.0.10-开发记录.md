# 0.0.10 开发记录
## 1. 版本目标

围绕正交模式真实投影路径接入，先打通数学工具、相机对象与控制器状态链，使 `ProjectionMode::orthographic` 不再停留在占位状态，为后续 UI 与 `Lens Zoom` 评审提供可验证基础。

## 2. 实际完成内容

### 2.1 补齐正交投影矩阵数学工具

- 在 `scene_contract::math` 中新增 `makeOrthographic(...)`
- 保持现有透视矩阵与其他数学工具行为不变

### 2.2 Camera 接入真实正交投影输出

- `Camera` 新增：
  - `ProjectionMode`
  - `setProjectionMode(...)`
  - `setOrthographicHeight(...)`
- `Camera::buildCameraData()` 现在根据模式输出：
  - 透视矩阵
  - 正交矩阵

### 2.3 打通 OrbitCameraController 与最小验证入口

- `OrbitCameraController` 现在会将：
  - `projectionMode`
  - `orthographicHeight`
  真正传给 `Camera`
- `ProjectionMode::orthographic` 不再回退到透视路径
- `Viewport` 新增：
  - `setProjectionMode(...) / projectionMode()`
  - `setOrthographicHeight(...) / orthographicHeight()`
- `Camera` 面板新增最小验证入口：
  - `Projection`
  - `Ortho Height`

- 本版本未新增阻塞问题归档项

## 6. 已知问题与限制

- 正交模式当前只完成真实投影路径接入，尚未扩展完整交互策略
- `Focus All / Focus Point / Focus Sphere` 在正交模式下仍未做专门 framing 优化
- `Lens Zoom` 仍未进入实际实现

## 7. 本版本遗留项

- 后续需要评审正交模式下的缩放语义与 framing 策略
- `Lens Zoom` 的行为边界与 UI 方案仍需单独评审
- 如需继续推进，应先明确正交模式下：
  - 默认缩放模式
  - 聚焦策略
  - 裁剪更新策略

## 8. 下一版本建议

建议下一版本优先讨论：

1. 正交模式下的缩放与聚焦行为
2. `Lens Zoom` 的行为边界
