# 0.0.7 开发计划

## 1. 版本目标

在已有 `worldBounds / sceneBounds / orbitCenter / Focus Point / Focus All` 最小接口的基础上，正式将聚焦能力接入测试界面，并补齐基础的距离拟合与裁剪参数更新逻辑，形成可验证的聚焦闭环。

## 2. 本版本范围

### 2.1 包含内容

- 在 `test_viewer` 中新增聚焦计算工具
- 将 `Focus All` 接入测试界面
- 将 `Focus Point` 接入测试界面，并提供世界坐标输入
- 聚焦动作时联动更新基础 `distance`
- 聚焦动作时联动更新基础 `near/far`
- 将现有 `Focus Sphere` 收口到统一聚焦链路

### 2.2 不包含内容

- 屏幕拾取
- 选中系统
- `Focus Selected`
- 每帧自动 `near/far` 更新
- 视锥裁剪
- 调试绘制
- 正式引入 `BoundingSphere` 共享协议类型

## 3. 已确认的设计约束

### 3.1 聚焦策略

- `Focus All`：自动重算相机距离
- `Focus Point`：第一版保持当前距离不变
- `near/far`：只在聚焦动作时更新
- `Focus Point`：提供世界坐标输入

### 3.2 空间约定

- `orbitCenter` 一律是世界空间点
- `Focus Point` 输入为世界空间坐标
- `sceneBounds` 使用世界空间 `Aabb`

### 3.3 Bounds 策略

- 当前版本继续以 `Aabb` 为基础范围数据
- 如需拟合球半径，仅从 `Aabb` 临时派生
- 不在本版本中引入独立 `BoundingSphere` 协议类型

## 4. 分步骤执行计划

### 步骤 1：补聚焦计算工具与基础裁剪更新

目标：

- 在 `test_viewer` 中新增聚焦计算工具模块
- 基于 `Aabb` 计算：
  - bounds center
  - 拟合距离
  - 聚焦动作使用的基础 `near/far`
- 为 `Focus All` 与 `Focus Point` 提供统一计算入口

完成判定：

- 已有独立聚焦计算 helper
- `Focus All` 和 `Focus Point` 可复用同一套计算入口
- 当前工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 2

### 步骤 2：将 Focus All / Focus Point 接入测试界面

目标：

- 在 `Actions` 区域增加 `Focus All`
- 在相机控制区域增加 `Focus Point` 世界坐标输入与触发按钮
- 聚焦动作触发后同步更新：
  - `orbitCenter`
  - `distance`
  - `near/far`
- 保持现有旋转、平移、缩放行为不回归

完成判定：

- 用户可从测试界面触发 `Focus All`
- 用户可输入世界坐标并触发 `Focus Point`
- 控制面板状态与视口状态保持一致

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 3

### 步骤 3：统一 Focus Sphere 路径并完成最小闭环

目标：

- 将现有 `Focus Sphere` 改为走统一聚焦链路
- 清理聚焦相关的分散逻辑
- 验证 `Focus All / Focus Point / Focus Sphere` 三条路径共用同一套基础策略

完成判定：

- 三种聚焦动作行为一致且状态同步正常
- 当前版本形成完整的最小聚焦闭环
- 当前工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后本版本开发完成

## 5. 风险与注意事项

- `Focus Point` 第一版不自动改距离，避免“点无尺寸”导致距离策略失真
- `near/far` 只在聚焦动作时更新，避免当前版本引入每帧投影抖动
- 若开发过程中发现 `Aabb` 语义不完整但不阻塞当前步骤，应登记到 `ReviewDoc/`
