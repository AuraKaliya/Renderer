# 0.0.8 开发计划

## 1. 版本目标

围绕当前轨道相机的缩放与裁剪问题，补齐透视模式下 Dolly 缩放过程中的基础 `near/far` 联动，并建立后续扩展正交模式与多种缩放模式所需的状态接口基础。

## 2. 本版本范围

### 2.1 包含内容

- 拆分相机裁剪计算逻辑，新增独立裁剪计算 helper
- 在透视模式下补齐 Dolly 缩放过程中的基础 `near/far` 联动
- 为相机控制器增加：
  - `ProjectionMode`
  - `ZoomMode`
  的状态建模
- 保持当前测试界面与聚焦链路行为不回归

### 2.2 不包含内容

- 正交模式的实际渲染接入
- Lens Zoom 的完整行为实现
- 正交模式与透视模式的 UI 切换
- 视锥裁剪
- 拾取与选中
- `Focus Selected`

## 3. 已确认的设计约束

### 3.1 缩放主模式

- 当前主缩放模式继续保持 `Dolly`
- 不将主缩放模式切换为“固定相机位置，仅 FOV 缩放”

### 3.2 逻辑拆层

- `camera_focus_utils` 负责聚焦与 framing
- `camera_clip_utils` 负责 `near/far` 计算
- 后续如需扩展，可再引入 `camera_zoom_utils`

### 3.3 状态建模

- `ProjectionMode` 与 `ZoomMode` 必须分离建模
- 本版本只完成状态接口，不实现正交渲染行为

## 4. 分步骤执行计划

### 步骤 1：拆分相机裁剪计算逻辑

目标：

- 新增独立 `camera_clip_utils`
- 将当前聚焦 helper 中的 `near/far` 计算逻辑抽离
- 保持现有 `Focus All / Focus Point / Focus Sphere` 行为不回归

完成判定：

- `near/far` 计算不再内嵌在聚焦 helper 中
- `camera_focus_utils` 与裁剪逻辑边界清晰
- 当前工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 2

### 步骤 2：补齐透视模式下 Dolly 缩放的 near/far 联动

目标：

- 在滚轮缩放路径中联动更新基础 `near/far`
- 降低近距离观察时的裁剪异常
- 保持当前旋转、平移、聚焦行为不回归

完成判定：

- 缩放时 `near/far` 会同步更新
- 当前 `Camera` 面板中的 `Near / Far` 会正确回填
- 当前工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 3

### 步骤 3：建立 ProjectionMode / ZoomMode 状态接口

目标：

- 为 `OrbitCameraController` 增加：
  - `ProjectionMode`
  - `ZoomMode`
- 完成状态读写接口
- 保持当前默认行为仍为透视模式 + Dolly 缩放
- 为后续正交模式和 Lens Zoom 扩展预留入口

完成判定：

- 相机控制器已具备投影模式与缩放模式状态接口
- 当前默认行为不发生功能回归
- 当前工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后本版本开发完成

## 5. 风险与注意事项

- 本版本只做透视模式下的实际缩放联动，不提前接入正交渲染
- 若在实现过程中发现正交参数模型需要提前落位，但不阻塞当前步骤，应登记到 `ReviewDoc/`
- 缩放联动裁剪后，需特别关注极近距离下的 `near/far` 是否出现跳变过大

## 6. 版本完成后的收尾要求

当本版本全部步骤完成后，需要在 `VersionDoc` 下生成：

- `0.0.8-开发记录.md`

并按 `ANGENT.md` 规则执行版本提交。
