# 0.0.4-开发计划
## 1. 版本目标

将当前基于参数面板的相机控制扩展为“视口直接交互”的相机控制链路，为后续视角旋转、旋转中心变更、视口缩放与更完整的编辑器交互打基础。

## 2. 本版本范围
### 2.1 包含内容

- 扩展 `OrbitCameraController`，补齐 `rotate / zoom / pan` 动作接口
- 在 `Viewport` 中接入鼠标旋转与滚轮缩放
- 在 `Viewport` 中接入平移操作并同步控制面板状态

### 2.2 不包含内容

- 拾取选点与基于命中的旋转中心切换
- 正交相机与透视/正交切换
- 相机状态持久化
- 更复杂的编辑器导航模式

## 3. 设计原则

### 3.1 责任边界

- `Camera` 负责基础相机数据与矩阵输出
- `OrbitCameraController` 负责交互动作与相机状态约束
- `Viewport` 负责 Qt 事件接入与渲染刷新
- `ViewerWindow` 负责把视口状态同步到控制面板

### 3.2 非侵入扩展

- 不破坏现有控制面板参数接口
- 不让控制面板直接操作 OpenGL 后端
- 不把 Qt 事件逻辑下沉到渲染库模块

## 4. 分步骤执行计划
### 步骤 1：扩展 OrbitCameraController 的交互动作能力

目标：
- 增加 `rotate / zoom / pan` 接口
- 增加 yaw、pitch、distance 的约束与归一化处理
- 为后续视口事件接入提供稳定的控制器能力层

完成判定：
- `OrbitCameraController` 已具备交互动作接口
- 当前面板驱动的相机行为不回归
- 当前版本编译通过

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 2

### 步骤 2：接入左键拖拽旋转与滚轮缩放

目标：
- 在 `Viewport` 中接入鼠标左键拖拽旋转
- 在 `Viewport` 中接入滚轮缩放
- 将交互后的相机状态同步回控制面板

完成判定：
- 用户可直接在视口内旋转视角
- 用户可通过滚轮改变观察距离
- 控制面板中的相机参数能反映视口交互结果

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 3

### 步骤 3：接入平移操作并收口交互细节

目标：
- 在 `Viewport` 中接入平移操作
- 完成交互状态收口，如鼠标释放、离开窗口后的状态复位
- 保证 `Reset Defaults` 与 `Focus Sphere` 与交互链路兼容

完成判定：
- 用户可直接在视口内平移观察中心
- 控制面板与预设动作不会被交互状态破坏
- 当前版本交互链路完整可用

测试与构建：

- 由用户执行
- 用户确认通过后本版本开发完成
