# 0.0.15 开发记录
## 1. 版本目标

围绕 `Viewport Zoom` 作为“局部工作窗口缩放”的语义，继续扩展 `test_viewer` 的视口交互能力，使其更接近 Blender 一类 DCC 软件的工作方式：以鼠标位置作为缩放锚点，通过调整当前相机工作中心并复用现有缩放路径，建立一个缩放后仍可继续进行旋转、平移与后续缩放的可工作窗口。

## 2. 实际完成内容

### 2.1 建立 Viewport Zoom 的最小状态与结构落点

- 新增：
  - `src/test_viewer/src/viewport_zoom_state.h`
  - `src/test_viewer/src/viewport_zoom_state.cpp`
- 为 `Viewport Zoom` 引入最小状态结构，当前保留：
  - `AnchorMode`
  - `ZoomOperationKind`
  - 最近一次锚点采样
  - 是否处于激活状态
- `Viewport` 中新增：
  - `viewportZoomState_`
  - `refreshViewportZoomState()`
- 在以下已有相机状态变化路径后同步 `Viewport Zoom` 的最小状态：
  - projection mode 变化
  - zoom mode 变化
  - orbit center 变化
  - focus / preset / reset
  - pan

### 2.2 接入 Blender 风格的最小 Viewport Zoom 闭环

- 在 `Viewport` 中新增：
  - `sampleViewportZoomAnchor(...)`
  - `applyViewportZoom(...)`
- `sampleViewportZoomAnchor(...)` 当前基于：
  - 鼠标位置
  - 当前相机姿态
  - 经过当前 `orbitCenter` 的工作平面
  计算鼠标锚点
- `applyViewportZoom(...)` 形成最小闭环：
  1. 采样缩放前锚点
  2. 执行当前相机缩放路径
  3. 采样缩放后锚点
  4. 用前后差值补偿 `orbitCenter`
- 当前将 `Ctrl + 滚轮` 作为 `Viewport Zoom` 的最小验证入口
- 普通滚轮继续保留原有：
  - `Perspective + Dolly`
  - `Perspective + Lens`
  - `Orthographic + Lens`
  缩放语义

### 2.3 明确持续工作中心采用 orbitCenter 语义

- 在实际交互验证过程中，发现如果将 `Viewport Zoom` 的工作中心长期保存在独立 `workspaceCenter` 中，而旋转仍只消费 `orbitCenter`，会出现：
  - 首次 `Ctrl + 滚轮` 后工作中心变化
  - 旋转后再次 `Ctrl + 滚轮`，新的工作中心不能继续自然变化
- 结合本版本讨论结论与经典建模软件导航范式，当前第一版明确采用：
  - `Viewport Zoom` 的持续工作中心落回 `orbitCenter`
- 相应调整为：
  - 鼠标锚点取样工作平面中心直接采用当前 `orbitCenter`
  - `viewport_zoom_state` 不再承担长期工作中心存储职责
  - `workspaceCenter` 相关状态与 helper 从最小状态结构中移除

### 2.4 与 Focus / Reset 的边界收口

- 以下行为会显式重置当前 `Viewport Zoom` 工作状态：
  - `resetDefaults()`
  - `applySphereFocusPreset()`
  - `focusOnPoint()`
  - `focusOnScene()` / `applyFocusBounds()`
- 这样 `Viewport Zoom` 与：
  - `Focus Point`
  - `Focus All`
  - 预设 framing
  的边界更清晰：
  - `Focus` 属于 framing
  - `Viewport Zoom` 属于局部工作窗口建立

## 6. 本版本问题修复

- 在最小闭环接入后，发现 `Viewport Zoom` 若长期依赖独立 `workspaceCenter`，而旋转仍只消费 `orbitCenter`，会导致：
  - 一次 `Ctrl + 滚轮` 后中心变化
  - 旋转后再次 `Ctrl + 滚轮` 时，工作中心不能继续自然演进
- 本版本已修正为：
  - 将持续工作中心重新统一到 `orbitCenter` 语义
  - 使 `Ctrl + 滚轮 -> 旋转 -> Ctrl + 滚轮` 的连续交互具备更合理的结构基础

## 7. 已知问题与限制

- 当前 `Viewport Zoom` 的最小入口仍采用 `Ctrl + 滚轮`，尚未形成更完整的 UI 暴露或可配置入口
- 当前鼠标锚点仍基于：
  - 鼠标射线与经过 `orbitCenter` 的工作平面交点
  尚未接入对象真实命中或表面深度
- 当前第一版仍然复用现有相机缩放路径，尚未引入更完整的：
  - virtual target
  - tumble pivot
  - 工作空间系统
- `viewport_zoom_state` 目前仍是最小辅助状态，主要服务于当前交互闭环，不是后续完整导航系统的最终形态
