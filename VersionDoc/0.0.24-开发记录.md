# 0.0.24-开发记录

## 本轮目标

- 为测试展示 UI 增加 Qt 无关的适配层，降低 `Viewport`、参数化模型与控制面板之间的耦合。
- 将右侧控制面板按经典编辑器习惯拆成 `Scene / Inspector / Camera / Debug` 功能区。
- 为新增适配接口补充以打印和值确认为主的 smoke tests。
- 为后续参数化建模模型类型拓展准备 schema 驱动的参数字段入口。

## 完成内容

### 1. UI 状态适配层

- 新增 `viewer_ui_types.h`，定义控制面板可消费的对象、feature、unit、输入、节点用法、构造链接、派生参数、诊断与求值摘要状态。
- 新增 `ViewerUiStateAdapter`，从 `ParametricObjectDescriptor` 和展示属性生成 `SceneObjectUiState`。
- `ViewerWindow::Viewport::controlPanelState()` 改为先构建 UI state，再转换成现有 `ViewerControlPanel::PanelState`。

### 2. 参数化 schema 适配层

- 新增 `ParametricUiSchemaAdapter`，基于 `ParametricConstructionSchema` 输出 feature 参数字段组。
- 字段覆盖 `node / float / integer / vector3 / enum`，并携带语义、标签、节点 id、节点坐标和值。
- 已覆盖 Box、Cylinder、Sphere、Mirror、Linear Array 现有构造接口。
- `PanelState` 增加 feature schema 字段组，Inspector 新增 `Feature Parameters` 只读展示区。

### 3. 控制面板功能区重构

- 控制面板页签调整为：
  - `Scene`：对象、feature、node 选择，feature 增删启停，灯光。
  - `Inspector`：节点坐标、当前 feature schema 字段、对象/primitive/operator 参数。
  - `Camera`：相机参数、视图动作、模型变化视图策略。
  - `Debug`：求值摘要、overlay、bounds、unit、输入、节点用法、构造链接、派生参数、诊断与重置。
- `ViewerControlPanel` 构造函数拆分为 `buildScenePage / buildInspectorPage / buildCameraPage / buildDebugPage`，并抽出 `connectObjectInspectorSignals()`。

### 4. 接口测试

- 新增 `viewer_ui_state_adapter_smoke_tests`：
  - Box center-size 对象状态。
  - Box center-corner 派生参数。
  - invalid sphere clamp diagnostic。
- 新增 `parametric_ui_schema_adapter_smoke_tests`：
  - Box center-size schema 字段和值。
  - Cylinder axis endpoints schema 字段和值。
  - Mirror / Linear Array operator schema 字段和值。
- 新增 `parametric_model_add_adapter_smoke_tests`：
  - 通过添加面板输入生成 Box center-corner descriptor。
  - 通过添加面板输入生成 Cylinder axis-endpoints descriptor。
  - 通过添加面板输入生成 Sphere diameter descriptor。

### 5. 添加模型操作区

- `Scene` 页顶部新增独立 `Add Model` 操作区。
- `Scene Hierarchy` 不再承载快速添加按钮，只负责对象、feature、node 的选择与管理。
- `Add Model` 使用 Box / Cylinder / Sphere 分页表单：
  - Box 支持 `Center + Size`、`Center + Corner Point`、`Two Corner Points`。
  - Cylinder 支持 `Center + Radius + Height`、`Center + Radius Point + Height`、`Axis Endpoints + Radius`。
  - Sphere 支持 `Center + Radius`、`Center + Surface Point`、`Diameter Points`。
- 新增 `ParametricModelAddAdapter`，把添加表单输入转换为 `ParametricObjectDescriptor`，并自动挂载默认 Mirror 与 Linear Array operator feature。

### 6. 问题修正

- 新增单元的旋转中心改为跟随参数化 primitive 的当前 pivot：
  - Center 构造模式直接使用 center point。
  - 两点构造模式使用两点中点。
  - 参数修改进入 `applyParametricObjectDescriptor()` 时会重新计算 `metadata.pivot`。
  - 渲染 transform 每帧从当前 descriptor 解析 pivot，并围绕该 pivot 旋转/缩放。
- `Add Model` 内部不再使用 Box/Cylinder/Sphere 分页，改为“Model Type 下拉选择 + 动态参数面板”，为后续更多模型类型留出扩展空间。
- 默认场景对象、新增对象和 Sphere Focus 预设的旋转速度均改为 `0.0`，需要用户在 Inspector 中手动设置旋转速度后才会旋转。
- `Debug / Units` 从只读列表改为可选择列表，点击 unit 会同步选中其对应 feature，并刷新 Scene feature 列表与 Inspector 参数区，减少 feature/unit 选择指向不一致的问题。
- 渲染窗口鼠标单击命中模型时，会同时选中该对象的 primitive unit，不再保留之前选中的 operator/unit，避免从视口单选后右侧单元状态停留在旧 feature。
- `Debug` 页新增 `Operation Log` 输出区，支持按 `Selection / Picking / Panel / Model / Camera` 操作类型筛选，并提供清空按钮。
- 视口鼠标单选会记录点击释放、pick 射线、每个对象的 AABB hit/miss 与 world bounds、mesh 命中情况、最终 picked index，以及 selection state 的前后变化，便于定位单元选取不准确的原因。
- `Operation Log` 输出控件由列表改为只读文本编辑框，刷新时按行追加，便于复制和查看连续日志。
- 右侧控制面板宽度调整为 `420-600`，并让 Scene 页 `Add Model` 表单字段可伸展且长行可换行，缓解控件被遮挡的问题。
- 曾尝试“当前已选对象再次命中时切换到下一命中对象”的策略，但该策略不能解释 AABB/mesh 判定不一致的问题，已撤回，继续保留严格最近 mesh 命中选择。

## 后续建议

- 将 `Feature Parameters` 从只读列表升级为动态编辑控件，逐步替换手写的 Box/Cylinder/Sphere 参数面板。
- 增加新的模型类型时，优先扩展 `ParametricConstructionSchema` 与 `ParametricUiSchemaAdapter`，再接入具体求值器。
- 下一阶段可把 UI signal 收敛到统一 command/patch 入口，为 undo/redo 和批量参数修改做准备。
