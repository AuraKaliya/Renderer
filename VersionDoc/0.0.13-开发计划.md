# 0.0.13 开发计划

## 1. 版本目标

围绕 `Zoom Mode` 的完整 UI 策略，继续整理测试界面的相机面板与窗口层职责边界，使当前相机 UI 能明确表达：

- 当前投影类型
- 当前缩放语义
- 当前主参数 / 次参数 / 无效参数

同时收紧 `CameraControlWidget` 与 `ViewerWindow` 的职责，为后续接入 `Viewport Zoom` 与模型参数变化策略预留更清晰的结构入口。

## 2. 本版本范围

### 2.1 包含内容

- 固化 `Projection + Zoom Mode + Active Parameter` 的 UI 规则
- 继续拆解 `CameraControlWidget` 内部职责
- 收口 `ViewerWindow` 中的 UI 同步/编排职责
- 为后续 `camera_mode_ui_policy` 或同类规则层预留最小落点

### 2.2 不包含内容

- `Viewport Zoom`
- 模型参数变化后的：
  - 保持当前视角
  - 自动适配视图
  策略实现
- `Orthographic + Dolly` 的 UI 开放
- 新的渲染后端或协议层改动

## 3. 输入来源

本版本计划吸收以下评审输入：

- `ReviewDoc/0.0.13-评审遗留项.md`
- `TalkDoc/2026-03-12-ZoomMode完整UI策略-讨论.md`
- `TalkDoc/2026-03-12-UI层代码结构是否继续拆解-讨论.md`

本版本明确不吸收以下后续输入：

- `Viewport Zoom`
- 模型参数变化后的视角策略

## 4. 已确认的设计约束

### 4.1 Zoom Mode UI 约束

- `Projection`、`Zoom Mode`、`Active Parameter` 必须在 UI 语义上分层表达
- 不允许只靠简单的 enable/disable 来承载全部语义
- `Distance` 在透视与正交模式下都不应被简单隐藏

### 4.2 当前支持策略

- `Perspective + Dolly` 继续作为默认透视主路径
- `Perspective + Lens` 继续作为可选缩放语义
- `Orthographic + Lens` 继续作为当前正交主路径
- `Orthographic + Dolly` 暂不开放到 UI

### 4.3 结构收口约束

- 不把新的相机规则继续直接堆入 `ViewerWindow`
- 不把 `Viewport Zoom` 混入当前 `Zoom Mode` UI
- 不把模型参数变化策略直接并入当前相机主面板

## 5. 分步骤执行计划

### 步骤 1：收口 Camera 面板的模式与参数规则

目标：

- 明确 `Projection / Zoom Mode / Active Parameter` 在 UI 中的结构关系
- 让当前不同模式下的主参数、次参数、无效参数表达清晰

完成判定：

- `CameraControlWidget` 中已形成清晰的模式区与参数区边界
- 不同模式下的参数表达与启停规则一致可读
- 工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 2

### 步骤 2：收口 ViewerWindow 的 UI 同步与编排职责

目标：

- 减少 `ViewerWindow` 直接处理控件规则和状态回填细节
- 集中整理 `Viewport -> Panel` 的相机状态同步逻辑

完成判定：

- `ViewerWindow` 的 UI 同步职责较当前更集中、更清晰
- 相机面板状态回填逻辑没有行为回归
- 工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 3

### 步骤 3：整理规则层输入并完成版本收口

目标：

- 将本版本形成的 UI 规则和结构结果整理为后续可复用输入
- 为下一阶段的 `Viewport Zoom` 与模型变化策略评审准备稳定边界

完成判定：

- 本版本开发记录可清晰说明 UI 结构与规则调整
- 后续版本具备清晰评审输入

测试与构建：

- 由用户执行
- 用户确认通过后本版本开发完成
