# 0.0.12 开发记录
## 1. 版本目标

围绕 `Perspective + Lens` 的行为边界，补齐透视模式下的可选镜头缩放路径，并保持当前默认透视导航、聚焦与裁剪语义稳定，为后续 `Zoom Mode` UI 策略评审提供可验证基础。

## 2. 实际完成内容

### 2.1 补齐 Perspective + Lens 的控制器行为

- `OrbitCameraController` 在：
  - `ProjectionMode::perspective`
  - `ZoomMode::lens`
  组合下，滚轮缩放现在只修改 `verticalFovDegrees`
- 不再修改相机真实距离 `distance`
- 当前透视默认主路径仍保持为 `Perspective + Dolly`

### 2.2 接入最小验证入口与状态回填

- `Camera` 面板中的 `Zoom Mode` 从只读展示调整为最小可切换下拉框
- 当前支持最小验证：
  - `Perspective + Dolly`
  - `Perspective + Lens`
- `Orthographic` 模式下仍不开放完整 `Zoom Mode` 切换，仅保持当前已确认行为
- `syncControlPanel()` 已能回填当前 `zoomMode`

### 2.3 整理后续评审输入

- 新增讨论归档：
  - `Perspective + Lens` 行为边界
  - 视口缩放与参数调整边界
  - 正交相机距离职责
- 更新 `ReviewDoc`：
  - 明确 `0.0.12` 吸收了哪些遗留项
  - 明确哪些内容继续后置：
    - `Viewport Zoom`
    - 模型参数变化策略
    - `Zoom Mode` 完整 UI

## 3. 本版本吸收的遗留项

- 吸收：
