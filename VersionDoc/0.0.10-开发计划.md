# 0.0.10 开发计划
## 1. 版本目标

围绕正交模式真实投影路径接入，先打通数学工具、相机对象与控制器状态链，使 `ProjectionMode::orthographic` 不再停留在占位状态，为后续 UI 与 `Lens Zoom` 评审提供可验证基础。

## 2. 本版本范围
### 2.1 包含内容

- 在 `math_utils` 中补齐正交投影矩阵构造工具
- 在 `Camera` 中接入：
  - `ProjectionMode`
  - `orthographicHeight`
  - 正交投影矩阵输出
- 在 `OrbitCameraController -> Camera -> Viewport` 打通正交状态链
- 提供最小可验证入口，确保当前默认行为仍保持透视模式

### 2.2 不包含内容

- `Lens Zoom` 行为实现
- 正交模式完整 UI 方案
- `render_core` / `render_gl` 协议层扩展
- 正交模式专属交互策略重构

## 3. 已确认的设计约束

### 3.1 协议层约束

- `scene_contract::CameraData` 保持不变
- 渲染后端继续只消费：
  - `view`
  - `projection`
  - `position`
- `ProjectionMode` 不下沉到协议层

### 3.2 正交参数语义

- `orthographicHeight` 表示正交视体在垂直方向上的总高度
- 正交模式宽度由 `aspectRatio` 推导
- 不复用 `verticalFovDegrees`

### 3.3 版本执行约束

- 默认行为继续保持：
  - `Perspective`
  - `Dolly`
- 本版本不引入 `Lens Zoom`
- 本版本中不扩展复杂 UI，只提供最小验证入口

## 4. 分步骤执行计划
### 步骤 1：补齐正交投影矩阵数学工具

目标：

- 在 `math_utils` 中新增正交投影矩阵构造函数
- 保持现有透视矩阵与其他数学工具行为不回归

完成判定：

- `math_utils` 具备独立的正交投影矩阵构造入口
- 现有工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 2

### 步骤 2：Camera 接入真实正交投影输出

目标：

- 在 `Camera` 中增加投影模式与正交高度状态
- `Camera::buildCameraData()` 根据模式输出透视或正交矩阵

完成判定：

- `Camera` 不再只有透视输出路径
- 默认透视行为保持不变
- 现有工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 3

### 步骤 3：打通 OrbitCameraController 与最小验证入口

目标：

- `OrbitCameraController` 将：
  - `projectionMode`
  - `orthographicHeight`
  真正传给 `Camera`
- 在测试界面中补充最小验证入口

完成判定：

- `ProjectionMode::orthographic` 真实生效
- 默认透视模式未回归
- 当前工程可正常编译

测试与构建：

- 由用户执行
- 用户确认通过后本版本开发完成

## 5. 风险与注意事项

- 正交模式接入后，`Focus All / Focus Point / Focus Sphere` 的 framing 行为可能与透视模式存在视觉差异，本版本以“投影路径打通”为主，不在本版扩展完整策略
- 不允许在本版本执行中途直接插入 `Lens Zoom`
- 若发现阻塞问题，按 `DebugDoc` 流程处理；若发现非阻塞疏漏，按 `ReviewDoc` 流程登记
