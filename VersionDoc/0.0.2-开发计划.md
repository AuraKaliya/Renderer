# 0.0.2-开发计划

## 1. 版本目标

将 `test_viewer` 当前集中在 `ViewerWindow::buildControlPanel()` 中的控制面板逻辑拆分为独立 Widget，降低 `ViewerWindow` 复杂度，并为后续继续扩展相机控制、光照控制和材质控制建立可维护的 UI 结构。

## 2. 本版本范围

### 2.1 包含内容

- 提取 `ViewerControlPanel`
- 提取 `SceneObjectControlWidget`
- 提取 `LightingControlWidget`
- 提取 `CameraControlWidget`
- 通过信号/槽将控制面板与 `ViewerWindow`、`Viewport` 连接

### 2.2 不包含内容

- 鼠标驱动的相机旋转和平移
- `TextureHandle` 接入
- 材质纹理编辑
- 参数面板样式美化
- 数据持久化

## 3. 设计原则

### 3.1 职责划分

- `ViewerWindow`
  - 负责主窗口布局
  - 负责连接 `Viewport` 与控制面板信号

- `ViewerControlPanel`
  - 负责组织各子控制组件
  - 负责暴露统一的 UI 信号

- `SceneObjectControlWidget`
  - 负责单个对象的显隐、旋转速度、颜色控制

- `LightingControlWidget`
  - 负责环境光与主光方向控制

- `CameraControlWidget`
  - 负责相机距离与 FOV 控制

### 3.2 耦合约束

- 子 Widget 不直接持有 `Viewport*`
- 子 Widget 仅负责界面输入与信号发出
- `ViewerWindow` 负责信号与 `Viewport` 方法的绑定

## 4. 分步骤执行计划

### 步骤 1：提取 ViewerControlPanel 骨架

目标：

- 新增 `ViewerControlPanel` 类
- 将 `ViewerWindow` 中右侧面板整体迁移到独立 Widget
- 保持当前界面行为不变

完成判定：

- `ViewerWindow::buildControlPanel()` 不再承担完整面板实现
- 面板仍可正常显示和工作

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 2

### 步骤 2：提取 LightingControlWidget 与 CameraControlWidget

目标：

- 将光照和相机控制从总面板中分离为独立组件
- `ViewerControlPanel` 负责装配这两个子组件

完成判定：

- 光照和相机部分不再直接堆叠在总面板实现中
- 信号/槽仍正常工作

测试与构建：

- 由用户执行
- 用户确认通过后进入步骤 3

### 步骤 3：提取 SceneObjectControlWidget

目标：

- 将单个对象控制抽象成可复用组件
- 用多个实例替代当前重复拼接逻辑

完成判定：

- 对象控制逻辑不再在总面板中重复展开
- 三个对象控制实例行为正确

测试与构建：

- 由用户执行
- 用户确认通过后本版本开发完成
