# 0.0.4-开发记录
## 1. 版本目标

将当前基于参数面板的相机控制扩展为“视口直接交互”的相机控制链路，为后续视角旋转、旋转中心变更、视口缩放与更完整的编辑器交互打基础。

## 2. 实际完成内容

### 2.1 扩展 OrbitCameraController 的交互动作能力
- 新增 `rotate / zoom / pan` 接口
- 增加 yaw 归一化与 pitch 夹紧逻辑
- 将相机位置计算收口为控制器内部能力

### 2.2 接入视口旋转与缩放交互
- 在 `Viewport` 中接入左键拖拽旋转
- 在 `Viewport` 中接入滚轮缩放
- 将交互后的相机状态同步回控制面板

### 2.3 接入平移与交互状态收口
- 在 `Viewport` 中接入右键拖拽平移
- 增加鼠标离开窗口与视口失焦时的交互状态复位
- 保持 `Reset Defaults` 与 `Focus Sphere` 与新交互链路兼容

### 2.4 问题修复与归档
- 修复“左键旋转时伴随缩放”的轨道相机距离语义错误
- 将问题归档到 `DebugDoc/0.0.4-问题.md`

## 4. 用户构建/测试结论

- 用户已完成构建与测试
- `0.0.4` 当前开发计划中的步骤已确认通过
- 视口旋转、缩放、平移交互已验证可用
- 旋转时伴随缩放的问题已确认关闭

## 5. 已知问题与限制

- 当前旋转中心 `orbitCenter` 仍然是隐式状态，控制面板中不可见
- 还没有引入模型包围盒、场景总包围盒和基于 bounds 的聚焦逻辑
- 近远裁剪面仍较为静态，尚未根据目标范围自动调整
- 尚未接入选中拾取、以选中对象为中心旋转等编辑器能力

## 6. 下一版本建议

建议后续版本优先补齐以下能力：

- 共享层中的包围盒与场景空间数据协议
- `orbitCenter` 的显式展示与调试能力
- `Focus All / Focus Selected / Focus Point`
- 基于包围盒的近远裁剪面自动更新
- 选中拾取与旋转中心切换逻辑
