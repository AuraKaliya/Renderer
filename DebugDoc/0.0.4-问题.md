# 0.0.4-问题
## 1. 问题描述

在 `0.0.4` 的视口交互相机开发过程中，左键按下拖动时出现“旋转视角的同时伴随缩放”的异常现象，导致交互体验不符合轨道相机预期。

## 2. 问题原因

`OrbitCameraController` 中相机位置计算使用了非标准轨道公式：

- `x/z` 使用 `distance` 作为水平半径
- `y` 使用 `tan(pitch) * distance`

这会导致 `distance` 实际表示“水平面半径”而不是“相机到观察中心的真实距离”。当 `pitch` 变化时，真实相机半径也会跟着变化，因此在旋转俯仰角时会产生明显的缩放感。

## 3. 解决方案

将轨道相机位置计算改为标准球坐标半径模型：

- `x = center.x + distance * cos(pitch) * sin(yaw)`
- `y = center.y + distance * sin(pitch)`
- `z = center.z + distance * cos(pitch) * cos(yaw)`

这样 `distance` 始终表示相机到观察中心的真实距离，旋转只改变方向，缩放只由 `zoom()` 控制。

## 4. 期望结果

- 左键拖拽旋转时不再出现缩放感
- 滚轮缩放行为保持正常
- `Distance` 参数与真实观察距离语义一致
- `Reset Defaults` 与 `Focus Sphere` 等现有预设行为不回归
