# 0.0.8-问题

## 1. 问题描述

在当前版本中，重复点击 `Focus All` 或重复点击 `Focus Sphere` 时，场景内物体显示不稳定。具体表现为：

- `distance` 数值会变化
- `near/far` 数值会变化
- 同一聚焦动作多次触发后，画面结果不一致

## 2. 问题原因

当前聚焦逻辑直接使用了对象“动画瞬间”的 `world AABB` 作为聚焦范围输入。

由于场景中的对象持续旋转：

- `box`
- `cylinder`
- `sphere`

它们的瞬时 `world AABB` 会随旋转角度变化。重复点击聚焦时，输入范围本身发生变化，因此聚焦结果中的：

- `distance`
- `near/far`

也会出现抖动和不一致。

## 3. 解决方案

将聚焦范围改为“稳定聚焦 bounds”：

- 不再直接使用旋转后的瞬时 `world AABB`
- 改为基于对象 `localBounds`
- 计算对象局部包围盒中心
- 结合当前变换求得世界中心
- 基于局部包围盒半径和当前最大轴缩放，派生稳定包围球半径
- 再转换为稳定的聚焦 AABB

同时：

- `Focus All` 改为聚合当前可见对象的稳定聚焦 bounds
- `Focus Sphere` 也改为走同一套稳定聚焦 bounds

## 4. 期望结果

- 重复点击 `Focus All` 时，`distance` 与 `near/far` 保持稳定
- 重复点击 `Focus Sphere` 时，`distance` 与 `near/far` 保持稳定
- 对象持续旋转时，聚焦结果不再随动画抖动
- 现有 `Focus Point`、显隐控制与相机交互不回归
