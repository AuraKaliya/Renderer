# 正交模式真实投影路径接入讨论

## 1. 讨论背景

当前项目已经在 `OrbitCameraController` 中预留了：

- `ProjectionMode`
- `orthographicHeight`

但真实渲染链路仍然只有透视投影路径：

- `Camera::buildCameraData()` 仅构造透视投影矩阵
- `OrbitCameraController::buildCameraData()` 在 `orthographic` 分支下仍回退到透视路径

因此，下一阶段需要优先讨论“正交模式真实投影路径接入”的落点与范围。

## 2. 当前实现检查结论

### 2.1 协议层是否需要变更

当前 `scene_contract::CameraData` 仅包含：

- `view`
- `projection`
- `position`

该结构对透视投影和正交投影都成立，因为渲染后端实际消费的是投影矩阵本身，而不是投影模式枚举。

结论：

- 正交模式真实投影路径接入不要求修改 `scene_contract::CameraData`
- 本阶段不应把 `ProjectionMode` 下沉到协议层
- 投影模式应继续停留在 `test_viewer` 的相机控制层

### 2.2 当前阻塞点

当前正交模式未真正生效的阻塞点主要有三个：

1. `math_utils` 中没有正交投影矩阵构造工具
2. `Camera` 没有正交投影参数与正交矩阵输出能力
3. `OrbitCameraController` 没有把 `ProjectionMode::orthographic` 真正映射到 `Camera`

### 2.3 当前不应提前做的部分

以下内容不建议与“正交模式真实投影路径接入”混做：

- `Lens Zoom` 行为实现
- 投影模式 UI 复杂化
- 正交模式下完整交互体验重构
- `render_core` / `render_gl` 协议扩展

原因：

- 正交模式第一阶段只需要打通真实投影矩阵路径
- 其余内容会放大改动范围，增加回归风险

## 3. 设计结论

### 3.1 投影模式落点

建议保持当前分层：

- `OrbitCameraController`
  - 负责持有 `ProjectionMode`
  - 负责持有 `orthographicHeight`
- `Camera`
  - 负责根据模式输出真实投影矩阵
- `scene_contract::CameraData`
  - 只承载最终 `view/projection`

### 3.2 正交参数设计

正交模式必须使用独立参数：

- `orthographicHeight`

不能复用：

- `verticalFovDegrees`

建议语义：

- `orthographicHeight` 表示正交视体在垂直方向上的总高度
- 宽度由 `aspectRatio` 推导

### 3.3 最小实现路径

建议按以下最小链路接入：

1. 在 `math_utils` 增加 `makeOrthographic(...)`
2. 在 `Camera` 中增加：
   - `setProjectionMode(...)`
   - `setOrthographicHeight(...)`
3. 在 `Camera::buildCameraData()` 中根据模式输出：
   - 透视矩阵
   - 正交矩阵
4. 在 `OrbitCameraController::buildCameraData()` 中将：
   - `projectionMode_`
   - `orthographicHeight_`
   真正传给 `Camera`

## 4. 对 UI 的建议

正交模式真实投影路径接入的第一阶段，不建议直接同步做完整 UI 切换方案。

建议分两层：

1. 第一阶段
   - 先打通真实正交投影路径
   - 允许最小验证入口
   - 不急于开放复杂面板交互

2. 第二阶段
   - 再评审投影模式切换 UI
   - 再评审正交模式下的默认缩放策略
   - 再评审 `Lens Zoom`

## 5. 对 0.0.10 的建议

建议 `0.0.10` 聚焦以下内容：

1. `math_utils` 增加正交矩阵工具
2. `Camera` 接入真实正交投影输出
3. `OrbitCameraController -> Camera -> Viewport` 打通正交状态链
4. 提供最小可验证入口

明确不包含：

- `Lens Zoom`
- 正交模式完整 UI 方案
- `render_core` / `render_gl` 协议层扩展

## 6. 讨论结论

- 正交模式真实投影路径接入应作为 `0.0.10` 的优先主题
- 当前协议层不需要为了正交模式而改动
- 第一阶段目标应收敛为“真实正交投影矩阵打通”，不与 `Lens Zoom` 混做
