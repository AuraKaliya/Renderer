# 正交模式缩放聚焦与 Lens Zoom 讨论

## 1. 讨论背景

`0.0.10` 已完成正交模式真实投影路径接入，但当前缩放、聚焦、裁剪 helper 仍主要基于透视模式假设：

- `camera_focus_utils`
  - 通过 `verticalFovDegrees` 拟合 `distance`
- `camera_clip_utils`
  - 通过“相机到中心距离”估算 `near/far`
- `OrbitCameraController::zoom(...)`
  - 当前仅做 `distance` 变化

因此，需要先明确：

1. 正交模式下的缩放与聚焦语义
2. `Lens Zoom` 的行为边界

## 2. 当前实现检查结论

### 2.1 当前 helper 是透视导向的

当前 `Focus All / Focus Bounds` 的核心公式建立在：

- 目标范围半径
- 当前视口比例
- 透视 `FOV`

之上。

结论：

- 这套逻辑不能直接复用为正交模式的 framing 逻辑

### 2.2 当前 zoom 语义是 Dolly

当前 `zoom(...)` 的实现本质是：

- 改变相机真实距离 `distance`

这在透视模式下是合理的主缩放方式。

但在正交模式下，如果仍然只改 `distance`：

- 视图尺度不会像透视那样发生对应变化
- 用户会感知到“相机在动，但画面缩放语义不直观”

结论：

- 正交模式下不宜把 `Dolly` 作为默认主缩放语义

## 3. 正交模式下的缩放与聚焦建议

### 3.1 正交模式的默认主缩放建议

建议：

- `Orthographic` 默认主缩放模式采用 `Lens Zoom`

具体含义：

- 不优先改相机距离
- 优先改 `orthographicHeight`

原因：

- 正交投影下，画面尺度主要由正交视体大小决定
- 改 `orthographicHeight` 才是最直观的“放大 / 缩小”

### 3.2 正交模式下是否仍允许 Dolly

建议：

- 允许 `Orthographic + Dolly`
- 但不作为默认模式

原因：

- 某些场景下，用户仍可能需要通过移动相机来调整相对深度位置
- 但这应属于可选行为，而不是默认缩放行为

### 3.3 正交模式下的 Focus All / Focus Bounds

建议：

- `orbitCenter = bounds.center`
- 不再用 `FOV` 推导 `distance`
- 改为根据 `Aabb` 的宽高与当前视口比例，拟合 `orthographicHeight`

建议公式：

1. 取 `bounds` 半尺寸：
   - `halfWidth`
   - `halfHeight`
2. 根据视口比例 `aspectRatio`，计算满足完整包络所需的正交高度
3. 再乘一层 padding

结论：

- 正交模式下的 `Focus All / Focus Bounds` 应改变的是：
  - `orbitCenter`
  - `orthographicHeight`
- 不应把“自动退远相机距离”当成主策略

### 3.4 正交模式下的 Focus Point

建议第一版采用：

- `orbitCenter = point`
- 保持当前 `orthographicHeight` 不变
- 仅更新 `near/far`

原因：

- 点没有天然尺寸
- 在正交模式下，缺少尺寸提示时不应自动改视体大小

## 4. 正交模式下的裁剪建议

### 4.1 near / far 的主要依据

建议：

- 正交模式下 `near/far` 仍依据相机位置与目标范围的深度关系计算

原因：

- 即使投影方式变为正交，深度裁剪仍然真实存在
- 裁剪问题与“是否透视”无关，而与观察体深度范围有关

### 4.2 当前 clip 逻辑是否要重写

结论：

- 不需要完全重写
- 但需要将透视模式和正交模式的 clip 策略接口显式分支

建议方向：

- `camera_clip_utils`
  - 保留公共接口
  - 内部分为：
    - `Perspective`
    - `Orthographic`

## 5. Lens Zoom 的行为边界

### 5.1 Lens Zoom 的统一定义

建议把 `Lens Zoom` 定义为：

- 改变投影视体大小
- 不改变相机真实位置

对应到不同模式：

- `Perspective + Lens`
  - 改 `verticalFovDegrees`
- `Orthographic + Lens`
  - 改 `orthographicHeight`

### 5.2 Dolly 的统一定义

建议把 `Dolly` 定义为：

- 改变相机真实距离 `distance`
- 不主动改变投影视体参数

对应到不同模式：

- `Perspective + Dolly`
  - 当前主模式
- `Orthographic + Dolly`
  - 可选模式，不作默认

### 5.3 是否允许四种组合

建议允许：

1. `Perspective + Dolly`
2. `Perspective + Lens`
3. `Orthographic + Dolly`
4. `Orthographic + Lens`

但分阶段实现：

- 先完成 `Orthographic + Lens`
- 最后再考虑 `Perspective + Lens`

原因：

- 正交模式先缺的是自然缩放语义
- 透视模式当前 `Dolly` 已经可用
- `Perspective + Lens` 更容易影响现有交互习惯，应后置

## 6. 对后续版本的建议

建议下一阶段按这个顺序推进：

1. 正交模式下的聚焦与缩放 helper 分支
2. 正交模式默认缩放改为 `Lens`
3. 正交模式的最小 UI 策略
4. 再单独评审 `Perspective + Lens`

## 7. 讨论结论

- 正交模式下默认主缩放建议采用 `Lens Zoom`
- 正交模式下的 `Focus All / Focus Bounds` 应以 `orthographicHeight` 拟合为主
- `Focus Point` 在正交模式下第一版建议只改中心，不自动改视体大小
- `Lens Zoom` 的统一定义应为“改变投影视体大小，不改相机真实位置”
- 四种投影/缩放组合可以保留，但应分阶段实现，不应一次性全接入
