# 0.0.9 评审优先级与相机结构讨论

## 1. 讨论背景

`0.0.8` 完成后，下一阶段的主要演进方向集中在三块：

- 代码结构优化
- 正交模式的真实投影路径接入
- `Lens Zoom` 的行为边界与 UI 方案

需要先明确评审优先级与依赖顺序，避免在实现阶段交叉返工。

## 2. 讨论内容

本次讨论按以下顺序展开：

1. 代码结构优化应先处理哪些边界
2. 正交模式真实投影路径应如何接入
3. `Lens Zoom` 的行为边界和 UI 方案应如何定义

## 3. 讨论结论

### 3.1 优先级顺序

建议按以下优先级推进：

1. 先做代码结构优化
2. 再做正交模式的真实投影路径接入
3. 最后再讨论并接入 `Lens Zoom`

### 3.2 结构优化的核心目标

- 显式拆分：
  - `render bounds`
  - `focus bounds`
- 将当前 `Viewport` 中的聚焦范围计算下沉到独立 helper
- 保持：
  - `camera_focus_utils`
  - `camera_clip_utils`
  - 后续可能的 `camera_zoom_utils`
  的职责边界清晰

### 3.3 正交模式接入前提

- 先完成相机结构整理
- 明确正交模式使用独立参数：
  - `orthographicHeight`
- 不复用透视模式的 `verticalFovDegrees`
- 正交模式接入时，`Camera` 需要真正支持正交投影矩阵构建

### 3.4 Lens Zoom 的定位

- `Lens Zoom` 不应先于正交模式和结构整理接入
- 它应建立在：
  - `ProjectionMode`
  - `ZoomMode`
  - 稳定的相机结构分层
  之上
- UI 上建议后置，不与当前主路径混在同一步版本中

## 4. 对后续版本的影响

- `0.0.9` 更适合聚焦结构优化
- 正交模式可作为 `0.1.0` 或后续单独版本目标
- `Lens Zoom` 更适合作为正交/透视双模式稳定后的再下一阶段能力

## 5. 后续建议

建议按以下阶段推进：

1. `0.0.9`
   - 整理 `render bounds / focus bounds`
   - 抽离聚焦范围 helper
   - 收紧 `Viewport` 内部职责
2. 下一版本
   - 接入正交投影矩阵
   - 接通正交模式状态到真实渲染链
3. 再下一版本
   - 评审并接入 `Lens Zoom`
   - 补充对应 UI 与交互策略
