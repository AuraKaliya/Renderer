# 正交相机距离职责讨论

## 1. 讨论背景

在当前正交模式真实投影路径和默认 `Lens` 缩放语义已经建立后，需要进一步确认：

- 正交相机是否还需要 `distance`
- 若需要，`distance` 的职责是什么

该问题会直接影响后续：

- 正交模式切换策略
- 裁剪更新策略
- 平移与旋转交互
- 模型参数变化后的视角保持

## 2. 讨论内容

### 2.1 正交相机是否仍需要 distance

结论：

- 需要

但它不再是正交模式下的主缩放参数。

### 2.2 正交模式下的主缩放参数

结论：

- 正交模式下的主缩放参数应为 `orthographicHeight`

原因：

- 正交投影下，画面尺度主要由正交视体大小决定
- `orthographicHeight` 才是自然的“放大 / 缩小”控制量

### 2.3 distance 在正交模式下的职责

建议明确为以下几类职责：

1. 相机在世界空间中的位置管理
2. 相机与 `orbitCenter` 的轨道关系维护
3. `near/far` 深度裁剪关系计算
4. 旋转、平移等交互的基础几何参考
5. 透视/正交切换时“保持当前视角”策略的一部分

### 2.4 distance 在正交模式下不应承担的职责

结论：

- 不应作为默认主缩放参数
- 不应与 `Lens Zoom` 混合承担同一种缩放语义

## 3. 讨论结论

- 正交相机需要 `distance`
- 但 `distance` 的职责应从“主缩放量”转为“视图位置与深度管理”
- 正交模式下默认主缩放参数继续保持为 `orthographicHeight`
- 后续设计中应明确区分：
  - `orthographicHeight`：视体大小
  - `distance`：位置、深度、轨道关系

## 4. 对当前版本的影响

- 不影响当前 `0.0.12` 的执行范围
- 本结论应作为后续“正交模式切换策略 / 视角保持 / 裁剪更新”评审输入

## 5. 后续建议

建议后续单独评审：

1. 透视切到正交时，默认是：
   - 保持当前视角
   - 还是自动适配视图
2. 正交模式下 `distance` 与 `near/far` 的更细粒度关系
3. 模型参数变化后，是否在保持当前视角前提下自动更新深度裁剪
