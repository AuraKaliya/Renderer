# Renderer 代码阅读指南

## 1. 阅读目标

阅读本项目代码时，建议不要一开始就从 OpenGL 细节入手，而是先建立“模块边界”和“数据流”认知，再下钻实现。

推荐阅读目标分为三层：

1. 先理解项目是怎么分层的
2. 再理解一帧数据是怎么流动的
3. 最后再理解 OpenGL 后端实现细节

## 2. 推荐阅读顺序

## 第一步：先看工程入口和模块边界

先读以下文件：

1. `CMakeLists.txt`
2. `src/scene_contract/CMakeLists.txt`
3. `src/render_core/CMakeLists.txt`
4. `src/render_gl/CMakeLists.txt`
5. `src/parametric_model/CMakeLists.txt`
6. `src/test_viewer/CMakeLists.txt`

阅读目标：

1. 明确工程由哪些目标组成
2. 明确依赖方向
3. 确认哪些模块可以独立演进

建议关注点：

1. `render_gl` 依赖 `render_core`
2. `parametric_model` 只依赖 `scene_contract`
3. `test_viewer` 同时依赖 `render_gl` 和 `parametric_model`

## 第二步：读共享数据协议

先读：

1. `src/scene_contract/include/renderer/scene_contract/types.h`
2. `src/scene_contract/include/renderer/scene_contract/math_utils.h`

阅读目标：

1. 弄清楚项目中“什么数据可以跨模块传递”
2. 理解 `MeshHandle`、`MaterialHandle` 的意义
3. 理解 `RenderableItem`、`FrameScene`、`RenderTargetDesc`
4. 理解当前矩阵约定和相机构造工具

如果这一层没读明白，后面会很容易把资源层、场景层和渲染层混在一起。

## 第三步：读 CPU 场景装配层

再读：

1. `src/render_core/include/renderer/render_core/scene_repository.h`
2. `src/render_core/src/scene_repository.cpp`
3. `src/render_core/include/renderer/render_core/frame_assembler.h`
4. `src/render_core/src/frame_assembler.cpp`

阅读目标：

1. 理解场景对象如何在 CPU 侧被保存
2. 理解对象如何更新 mesh/material/transform
3. 理解 `snapshot -> build -> FramePacket` 这条链

当前这里的实现不复杂，正因为简单，很适合先建立整体心智模型。

## 第四步：读参数化建模入口

再读：

1. `src/parametric_model/include/renderer/parametric_model/primitive_factory.h`
2. `src/parametric_model/src/primitive_factory.cpp`

阅读目标：

1. 理解参数化建模输出的最终形式是什么
2. 看清 `Box`、`Cylinder`、`Sphere` 是如何组织顶点和索引的
3. 建立“参数化几何 -> MeshData”的映射认知

建议阅读方法：

1. 先看 `makeBox`
2. 再看 `makeCylinder`
3. 最后看 `makeSphere`

因为这三个实现的复杂度是递增的。

## 第五步：读测试界面外壳

再读：

1. `src/test_viewer/src/main.cpp`
2. `src/test_viewer/src/viewer_window.h`
3. `src/test_viewer/src/viewer_window.cpp`

阅读目标：

1. 明确 Qt 是如何创建 OpenGL 上下文的
2. 明确离屏 FBO 是在哪里创建的
3. 明确测试对象什么时候上传 mesh 和 material
4. 明确每帧动画和场景更新在哪完成

建议重点跟踪这些函数：

1. `Viewport::initializeGL`
2. `Viewport::resizeGL`
3. `Viewport::paintGL`
4. `Viewport::rebuildFramePacket`

## 第六步：最后读 OpenGL 后端

最后读：

1. `src/render_gl/include/renderer/render_gl/gl_renderer.h`
2. `src/render_gl/src/gl_renderer.cpp`

阅读目标：

1. 理解 OpenGL 函数加载方式
2. 理解 shader 编译和 program 链接过程
3. 理解 mesh 缓存和 material 缓存
4. 理解真正的 draw 调用是怎样执行的

建议阅读顺序：

1. 先看 `GlRenderer` 对外接口
2. 再看 `MeshResource`、`MaterialResource`
3. 再看 `initialize`
4. 再看 `uploadMesh` / `uploadMaterial`
5. 最后看 `render`

不要一开始从 `render()` 读起，否则会被 OpenGL 调用细节打断整体理解。

## 3. 推荐的实际阅读过程方案

下面是一套适合 1 到 2 次代码阅读会议的过程方案。

### 阶段 A：建立总体认知

耗时建议：30 到 45 分钟

步骤：

1. 阅读 `00-文档索引.md`
2. 阅读 `01-项目总览与架构设计.md`
3. 阅读顶层 `CMakeLists.txt`
4. 画出项目依赖图

产出目标：

1. 你能说清楚每个模块负责什么
2. 你能说清楚哪些模块之间不能直接耦合

### 阶段 B：建立数据流认知

耗时建议：45 到 60 分钟

步骤：

1. 阅读 `types.h`
2. 阅读 `math_utils.h`
3. 阅读 `scene_repository.*`
4. 阅读 `frame_assembler.*`

产出目标：

1. 你能说清楚一帧数据从哪里来
2. 你能说清楚 `RenderableItem` 为什么只放 handle

### 阶段 C：建立功能模块认知

耗时建议：45 到 60 分钟

步骤：

1. 阅读 `primitive_factory.*`
2. 阅读 `viewer_window.*`
3. 跟踪初始化流程和每帧更新流程

产出目标：

1. 你能说清楚测试场景是如何构造的
2. 你能说清楚相机、动画、离屏渲染分别在哪实现

### 阶段 D：建立 OpenGL 后端认知

耗时建议：60 到 90 分钟

步骤：

1. 阅读 `gl_renderer.h`
2. 阅读 `gl_renderer.cpp`
3. 重点跟踪以下链路：
   - `initialize`
   - `uploadMesh`
   - `uploadMaterial`
   - `render`

产出目标：

1. 你能说清楚资源缓存如何工作
2. 你能说清楚为什么当前实现已经比“每帧上传全部数据”更合理

## 4. 推荐的阅读关注点

### 4.1 第一遍阅读重点

第一遍不要纠结 OpenGL 细枝末节，重点关注：

1. 分层是否清晰
2. 数据是否跨层污染
3. 资源是否通过句柄解耦
4. 流程是否符合“先装配，再统一渲染”

### 4.2 第二遍阅读重点

第二遍再重点看：

1. Mesh 上传路径
2. Material 上传路径
3. 顶点属性布局
4. shader 输入输出
5. 离屏 FBO 回显路径

### 4.3 第三遍阅读重点

如果后续要开始改代码，第三遍建议重点看：

1. 哪些地方适合扩 `TextureHandle`
2. 哪些地方适合加入批处理
3. `render_core` 该如何扩成更正式的渲染队列

## 5. 建议的阅读方法

1. 先按模块读，不要按文件名随机跳读
2. 每读完一层，自己复述一次“输入是什么，输出是什么”
3. 阅读 `render_gl` 时，始终回到 `FramePacket` 看它到底消费了什么
4. 阅读 `test_viewer` 时，始终区分“Qt 负责上下文”和“render_gl 负责绘制”

如果按这套顺序阅读，理解成本会明显低于直接从 `gl_renderer.cpp` 开始硬读。
