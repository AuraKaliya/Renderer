# Renderer 各模块功能说明

## 1. `scene_contract`

路径：

- `src/scene_contract/include/renderer/scene_contract/types.h`
- `src/scene_contract/include/renderer/scene_contract/math_utils.h`

### 1.1 作用

该模块是整个项目的数据协议层，负责定义所有跨模块共享的数据结构。

### 1.2 关键类型

#### 基础数学类型

- `Vec2f`
- `Vec3f`
- `Vec4f`
- `Mat4f`

这些类型是轻量 POD 风格数据，便于在不同模块之间直接传递。

#### 资源句柄

- `MeshHandle`
- `MaterialHandle`

它们是资源引用标识，用于让场景对象引用 GPU 资源，而不是直接持有大对象数据。

#### 渲染相关数据

- `VertexPNT`
- `MeshData`
- `MaterialData`
- `TransformData`
- `CameraData`
- `RenderableItem`
- `FrameScene`
- `RenderTargetDesc`

### 1.3 数学工具

`math_utils.h` 目前提供：

- 向量加减、点乘、叉乘
- 向量长度和归一化
- 单位矩阵、平移、缩放、旋转
- 矩阵乘法
- `makePerspective`
- `makeLookAt`

这些能力主要服务于测试界面和 CPU 侧变换生成。

## 2. `render_core`

路径：

- `src/render_core/include/renderer/render_core/frame_assembler.h`
- `src/render_core/include/renderer/render_core/scene_repository.h`
- `src/render_core/src/frame_assembler.cpp`
- `src/render_core/src/scene_repository.cpp`

### 2.1 作用

`render_core` 是 CPU 侧的场景装配层。

### 2.2 核心类

#### `SceneRepository`

职责：

1. 保存场景对象列表
2. 为对象分配 `ItemId`
3. 更新对象的 mesh 引用、material 引用和 transform
4. 按当前相机状态导出 `FrameScene`

当前接口：

- `add`
- `clear`
- `updateMeshHandle`
- `updateMaterialHandle`
- `updateTransform`
- `snapshot`

#### `FrameAssembler`

职责：

1. 接收 `FrameScene`
2. 生成 `FramePacket`
3. 为渲染器提供统一的一帧输入

当前实现很轻，只是直接把 `FrameScene` 转换为 `FramePacket`。

### 2.3 当前局限

当前 `render_core` 还没有：

1. 可见性裁剪
2. Opaque/Transparent 分层
3. 排序与批处理
4. Pass 系统

这部分是后续优化重点。

## 3. `render_gl`

路径：

- `src/render_gl/include/renderer/render_gl/gl_renderer.h`
- `src/render_gl/src/gl_renderer.cpp`

### 3.1 作用

`render_gl` 是 OpenGL 后端实现层。

### 3.2 对外接口

#### 生命周期

- `initialize`
- `shutdown`
- `isInitialized`
- `lastError`

#### Mesh 资源

- `uploadMesh`
- `updateMesh`
- `releaseMesh`

#### Material 资源

- `uploadMaterial`
- `updateMaterial`
- `releaseMaterial`

#### 渲染

- `render`

### 3.3 内部实现要点

#### OpenGL 函数加载

使用 `ProcResolver` 从当前 OpenGL 上下文中解析 OpenGL 函数地址。

#### Shader

当前使用内嵌 GLSL 字符串：

- 顶点着色器：处理位置和法线
- 片元着色器：处理基础颜色、定向光和环境光

#### Mesh 缓存

每个 mesh 对应一份 `MeshResource`，当前包含：

- `VAO`
- `VBO`
- `EBO`
- `indexCount`

#### Material 缓存

每个 material 对应一份 `MaterialResource`，当前仅保存：

- `MaterialData`

#### 当前渲染流程

1. 清空颜色、深度、模板
2. 设置相机矩阵
3. 设置灯光 uniform
4. 遍历 `FramePacket::opaqueItems`
5. 通过 `MeshHandle` 找到 `MeshResource`
6. 通过 `MaterialHandle` 找到 `MaterialResource`
7. 设置 `uWorld` 和 `uBaseColor`
8. 执行 `glDrawElements`

### 3.4 当前局限

1. 没有纹理系统
2. 没有透明物体处理
3. 没有批次合并
4. 没有 UBO/SSBO
5. 没有拾取、后处理、阴影

## 4. `parametric_model`

路径：

- `src/parametric_model/include/renderer/parametric_model/primitive_factory.h`
- `src/parametric_model/src/primitive_factory.cpp`

### 4.1 作用

负责参数化几何生成。

### 4.2 当前能力

- `makeBox`
- `makeCylinder`
- `makeSphere`

### 4.3 特点

1. 输出是纯 `MeshData`
2. 不涉及 OpenGL
3. 不依赖 Qt
4. 可直接作为未来建模库的入口雏形

### 4.4 当前局限

1. 仅有基础几何
2. 没有参数图、历史树或布尔运算
3. 没有边线、拓扑结构、CAD 精度建模能力

## 5. `test_viewer`

路径：

- `src/test_viewer/src/main.cpp`
- `src/test_viewer/src/viewer_window.h`
- `src/test_viewer/src/viewer_window.cpp`

### 5.1 作用

作为整个项目的验证界面，用于：

1. 创建 Qt OpenGL 上下文
2. 管理离屏 FBO
3. 初始化渲染器
4. 上传样例 mesh 和 material
5. 构造测试场景
6. 每帧更新变换并触发渲染

### 5.2 核心类

#### `ViewerWindow`

- 顶层主窗口
- 持有 `Viewport`

#### `ViewerWindow::Viewport`

- 继承自 `QOpenGLWindow`
- 管理 OpenGL 生命周期
- 管理三组测试对象

### 5.3 当前场景

当前内置 3 个演示对象：

1. 左侧蓝色 Box
2. 中间橙色 Cylinder
3. 右侧绿色 Sphere

它们具有不同的位置、缩放与旋转速度，用于验证：

- 多 mesh
- 多 material
- 多 transform
- 同帧统一渲染
