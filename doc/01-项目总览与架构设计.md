# Renderer 项目总览与架构设计

## 1. 项目目标

本项目是一个基于 `CMake + MinGW + OpenGL + Qt5` 的渲染库工程，当前目标是搭建一套可扩展的桌面渲染验证框架，满足以下要求：

1. 项目拆分为渲染库、参数化建模库、测试界面三部分。
2. 渲染库与参数化建模库之间不直接依赖。
3. 测试界面通过 `Qt OpenGL Window` 完成离屏渲染，并最终将结果以 texture 形式输出到窗口。
4. 渲染流程采用“资源先装配、渲染时统一提交”的思路，以降低每帧重复开销。

## 2. 当前工程结构

```text
Renderer/
├─ CMakeLists.txt
├─ src/
│  ├─ scene_contract/
│  ├─ render_core/
│  ├─ render_gl/
│  ├─ parametric_model/
│  └─ test_viewer/
└─ doc/
```

## 3. 模块分层

### 3.1 分层关系

```text
test_viewer
  |----> render_gl
  |        |----> render_core
  |                  |----> scene_contract
  |
  |----> parametric_model
             |----> scene_contract
```

### 3.2 设计原则

1. `scene_contract` 只定义共享数据协议，不包含 OpenGL、Qt 或复杂业务逻辑。
2. `render_core` 只负责场景组织和帧装配，不直接操作 GPU。
3. `render_gl` 是 OpenGL 后端，负责资源上传、缓存和真正的绘制执行。
4. `parametric_model` 负责几何生成，不知道 OpenGL 和 Qt 的存在。
5. `test_viewer` 是验证外壳，负责 OpenGL 上下文、离屏 FBO 和可视化展示。

## 4. 当前模块定位

### 4.1 `scene_contract`

- 项目中最基础的数据协议层。
- 为参数化建模和渲染提供统一的数据边界。
- 当前包含：
  - 顶点结构 `VertexPNT`
  - 几何数据 `MeshData`
  - 材质数据 `MaterialData`
  - 变换 `TransformData`
  - 相机 `CameraData`
  - 场景对象 `RenderableItem`
  - 句柄类型 `MeshHandle`、`MaterialHandle`

### 4.2 `render_core`

- 承担 CPU 侧场景管理与帧数据装配。
- 当前实现偏轻量，核心职责是：
  - 持有场景对象集合
  - 更新对象引用和变换
  - 生成渲染器可直接消费的 `FramePacket`

### 4.3 `render_gl`

- OpenGL 渲染后端。
- 当前已经实现：
  - OpenGL 函数加载
  - Shader 编译与 Program 链接
  - Mesh 缓存
  - Material 缓存
  - 最小定向光照
  - 按 `FramePacket` 执行绘制

### 4.4 `parametric_model`

- 参数化几何生成模块。
- 当前提供：
  - `Box`
  - `Cylinder`
  - `Sphere`

### 4.5 `test_viewer`

- Qt5 测试界面。
- 当前提供：
  - `QOpenGLWindow` 上下文
  - 离屏 `QOpenGLFramebufferObject`
  - 将离屏结果回拷到默认帧缓冲
  - 多物体场景展示

## 5. 当前数据流设计

### 5.1 高层数据流

```text
PrimitiveFactory 生成 MeshData
-> render_gl 上传为 MeshHandle
-> render_gl 上传 MaterialData 为 MaterialHandle
-> SceneRepository 保存 RenderableItem
-> FrameAssembler 生成 FramePacket
-> GlRenderer 执行绘制
-> test_viewer 将离屏结果显示到窗口
```

### 5.2 为什么使用 Handle

当前 `RenderableItem` 不直接持有大块几何和材质数据，而是通过句柄引用资源，原因是：

1. 避免每帧重复复制 `MeshData`
2. 避免渲染阶段重复上传 GPU 资源
3. 让场景层与资源层职责分离
4. 便于后续扩展纹理、shader variant、实例化等能力

## 6. 当前已实现能力

### 6.1 已完成

1. 工程骨架与模块拆分
2. `CMake` 模块组织
3. `Qt5` 测试窗口
4. 离屏 FBO 渲染并回显
5. Mesh/Material 资源缓存
6. 最小定向光照
7. 多物体、多材质、多 primitive 场景

### 6.2 当前偏验证性质的部分

1. `render_core` 仍是轻量实现，尚未加入可见性裁剪、排序和批处理
2. `render_gl` 还没有纹理系统和统一材质系统
3. `parametric_model` 还处于基础几何生成阶段
4. `test_viewer` 还没有参数面板和交互控件

## 7. 推荐的后续扩展方向

1. 在 `render_core` 增加渲染队列、排序和批次归并
2. 在 `render_gl` 增加 `TextureHandle`
3. 在 `render_gl` 中引入更完整的 `MaterialResource`
4. 在 `parametric_model` 增加更多建模算子
5. 在 `test_viewer` 中加入参数调整界面
