# Renderer 渲染与运行流程

## 1. 总体流程

项目当前的运行链路可以概括为：

```text
参数化建模
-> CPU 场景组织
-> GPU 资源上传
-> 帧数据装配
-> OpenGL 离屏渲染
-> texture 回显到窗口
```

## 2. 启动流程

### 2.1 程序入口

入口文件：

- `src/test_viewer/src/main.cpp`

入口职责：

1. 创建 `QApplication`
2. 创建 `ViewerWindow`
3. 进入 Qt 事件循环

### 2.2 主窗口创建

`ViewerWindow` 会创建 `Viewport`，并通过 `QWidget::createWindowContainer` 将其放入主窗口。

### 2.3 OpenGL 初始化

`Viewport::initializeGL()` 中完成：

1. Qt OpenGL 函数初始化
2. `GlRenderer::initialize()`
3. 上传测试场景用的 mesh
4. 上传测试场景用的 material
5. 在 `SceneRepository` 中注册场景对象

## 3. 资源准备流程

### 3.1 Mesh 生成

由 `PrimitiveFactory` 生成：

- Box
- Cylinder
- Sphere

输出类型为 `scene_contract::MeshData`。

### 3.2 Mesh 上传

`GlRenderer::uploadMesh()` 做的事：

1. 创建 `VAO`
2. 创建 `VBO`
3. 创建 `EBO`
4. 上传顶点数据
5. 上传索引数据
6. 配置顶点属性布局
7. 返回 `MeshHandle`

### 3.3 Material 上传

`GlRenderer::uploadMaterial()` 将 `MaterialData` 缓存到渲染器内部，并返回 `MaterialHandle`。

当前材质内容比较简单，仅包含 `baseColor`。

## 4. 场景装配流程

### 4.1 场景对象

每个对象由以下部分组成：

1. `MeshHandle`
2. `MaterialHandle`
3. `TransformData`

这些数据被包装为 `RenderableItem`。

### 4.2 SceneRepository

`SceneRepository` 是 CPU 侧场景容器。

职责：

1. 添加对象
2. 更新对象句柄
3. 更新对象变换
4. 用当前相机生成场景快照

### 4.3 FrameAssembler

`FrameAssembler` 将 `FrameScene` 和当前渲染目标尺寸打包成 `FramePacket`。

当前 `FramePacket` 包含：

1. `camera`
2. `opaqueItems`
3. `target`

## 5. 每帧更新流程

`Viewport::paintGL()` 的主要流程如下：

1. 依据当前时间计算动画角度
2. 更新 3 个对象的世界矩阵
3. 生成当前相机参数
4. 调用 `SceneRepository::snapshot()`
5. 调用 `FrameAssembler::build()`
6. 绑定离屏 FBO
7. 调用 `GlRenderer::render(framePacket_)`
8. 从离屏 FBO blit 到默认帧缓冲

## 6. 渲染执行流程

`GlRenderer::render()` 当前流程如下：

### 6.1 渲染前准备

1. 设置视口
2. 开启深度测试
3. 清除颜色、深度、模板缓冲
4. 绑定 shader program
5. 设置视图矩阵和投影矩阵
6. 设置光照 uniform

### 6.2 遍历绘制对象

对 `packet.opaqueItems` 中的每个对象：

1. 校验 `meshHandle`
2. 校验 `materialHandle`
3. 查找 `MeshResource`
4. 查找 `MaterialResource`
5. 绑定该对象的 `VAO`
6. 设置 `uWorld`
7. 设置 `uBaseColor`
8. 调用 `glDrawElements`

### 6.3 当前光照模型

当前使用最小光照模型：

1. 法线从顶点着色器传给片元着色器
2. 使用固定方向主光
3. 使用固定环境光强度
4. 以 `baseColor` 参与漫反射计算

## 7. 离屏渲染与 texture 输出

### 7.1 FBO

测试界面中的离屏目标使用 `QOpenGLFramebufferObject`。

当前用途：

1. 先把场景画到离屏纹理
2. 再把离屏结果复制到默认窗口帧缓冲

### 7.2 为什么这样设计

这与项目原始目标一致：

1. 渲染过程不直接耦合窗口默认缓冲
2. 最终产物是 texture
3. 后续更容易扩展为：
   - 截图导出
   - 后处理
   - 多通道渲染
   - 纹理复用

## 8. 当前实现与未来目标的关系

### 8.1 当前已满足

1. 模块解耦
2. 资源先上传、帧内统一提交
3. Qt5 离屏渲染并最终输出为 texture

### 8.2 未来仍可继续增强

1. `render_core` 增加队列与裁剪
2. `render_gl` 增加纹理和更完整材质
3. 引入更多 RenderPass
4. 在测试界面加入可调参数
